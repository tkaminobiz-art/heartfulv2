## 1. web-design-creation Part A との照合

### 1-1. 余白

| 条件 | 現状 | 判定 |
|------|------|------|
| セクション間 120〜200px | features 20px、cta-section 30px、trouble/reassurance 60px 等。**セクション間**は上下 padding の合計で 60+60=120 の箇所はOK。一方で features 20px + stats 16px 等は **約36px で不足** | ❌ 要修正 |
| 関連 8〜16px / 非関連 48〜80px | カード内・CTA内は 10〜18px 程度。セクション間の「ゆとり」が不足している箇所あり | △ |
| コンテンツ左右 10〜15% | max-width 1000〜1200px で中央寄せ。概ね準拠 | ○ |
| カード内 padding 32〜48px | 24px〜30px の箇所あり（cta-box 24px 30px、feature-box 18px 20px） | ❌ 要修正 |
| 見出しの上 > 見出しの下 | 未計測。trouble-title margin-bottom 40px 等、個別確認が必要 | △ |

### 1-2. タイポグラフィ

| 条件 | 現状 | 判定 |

| 和文 font-weight 600以上禁止 | **700・900 が多数**。header, feature-text, cta, stats, point-title, fv-cta-btn, voice-google-more-btn 等、全体で **30箇所以上** | ❌ 重大 |
| 数値 欧文・tabular-nums | 価格表・電話番号に tabular-nums の明示なし。欧文フォント指定なし | △ |

### 1-3. 色

| 条件 | 現状 | 判定 |

| 背景 オフホワイト・純白禁止 | **body background: #fff**、#header #fff、.grid-bg #fff、.points-content #fff、.cta-box #fff、.feature-box(1) #fff → **純白使用** | ❌ 禁止違反 |

## 6. 問題点一覧（修正優先度順）

### 重大（早急に修正）

1. **和文 font-weight 600 以上の使用**  
   - 見出し・リード・ボタン・バッジ等の **700・900 を 500 以下に変更**（太さは letter-spacing・font-size で補う）。
2. **純白背景 #fff の使用**  
   - body, header, .grid-bg, .points-content, .cta-box, .feature-box 等を **#FAFAF7 または #F5F5F0** に変更。
3. **同一レイアウトパターン 3 回以上**  
   - **point-item** 5 回 → 3 回に減らすか、2 種以上のレイアウト（例: 1 列テキストのみ、2 列、フル幅画像）に分ける。  
   - **CTA ボックス** 3 回 → 1〜2 回に減らすか、見た目（形・配置）を変える。
4. **見出し/本文 3 倍以上**  
   - FV 見出しを **48px 以上**（本文 16px なら 48px）、h2 を **36〜48px** にし、**3 倍** を満たす。

### 要修正

5. **アクセント 1 色**  
   - 緑とオレンジの両方を「アクセント」にしない。**メイン 1 色（緑）＋ CTA のみオレンジ**等、役割を分けて 80/15/5 に近づける。
6. **border-radius 999px（ピル型）**  
   - ボタンの **50px を 8〜12px** 程度に変更（角丸矩形）。
7. **セクション間余白 120〜200px**  
   - features, stats, cta-section の **上下 padding を増やし**、隣接セクションとの合計が 120px 以上になるようにする。
8. **カード内 padding 32〜48px**  
   - cta-box, feature-box の padding を **32px 以上**に。
9. **prefers-reduced-motion**  
   - アニメーションに `@media (prefers-reduced-motion: reduce) { ... }` で **無効化 or 短縮**を追加。
10. **画像 lazy loading**  
    - FV 以外の img に **loading="lazy"** を追加。

### 推奨（品質・一貫性）

11. **数値の tabular-nums**  
    - 電話番号・価格に `font-variant-numeric: tabular-nums;` を指定。
12. **画像アスペクト比**  
    - 一覧で 3:2 または 4:3 を統一し、CSS で aspect-ratio を指定。
13. **フォーカス可視化**  
    - `a:focus-visible`, `button:focus-visible` で **outline を消さず**、見やすいスタイル（2px 実線等）を指定。

## 7. ディレクター所見

- **良い点**: FV の左フル画像・右テキスト、ヘッダーのオマージュ構成、特徴 01/02/03 の意図的な差、実写写真の活用、Google 口コミ・「もっと見る」ボタンは、**設計意図が表れている**。土台はある。
- **足りない点**: 「美しさ」の**数値ルール**（余白・タイポ・色）と**禁止事項**の遵守が弱く、web-design-creation / design-system で定めた「Awwwards 8.0 を取れるか」の基準を**満たしていない**。特に**和文の font-weight** と**純白背景**・**同一パターン過多**は、第一印象とプロ品質の判断を下げる。
- **次の一手**: 上記「重大」「要修正」を**計画書に反映**し、実装（B）で一括修正。監視（C）は「デザイン仕様・数値一覧」と本採点表の禁止事項で**提出前チェック**を行う。
