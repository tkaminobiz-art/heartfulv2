# デザイン会議 — FV直下「Google口コミセクション」リニューアル

**対象**: FV直下の緑バー（#stats：お客様満足度・Google口コミ・作業写真）  
**現状評価**: 100点満点中おおよそ5点。リニューアルのため、問題の言語化と実装方針を会議で決定する。  
**日付**: 2026-03-13  
**参加**: デザイナーチーム、ディレクター

---

## 1. 現状の構成（何があるか）

- **背景**: セクション全体が単色の緑（`var(--green)`）。FV直下で幅いっぱい。
- **左ブロック**: オレンジの円（conic-gradientでリング状）内に「お客様満足度」「★5.0」の白文字。その右に🌿の装飾。
- **中央ブロック**: ラベル「Google口コミ」、その下に黄色の大きな「★5.0」、その下に「奈良・大阪・京都・兵庫・滋賀対応」の白文字。
- **右ブロック**: 作業写真（ソファを運ぶ様子）を横長で配置。緑地にそのまま貼り付け。

---

## 2. 何がおかしいか — 問題の言語化

### 2-1. 情報設計・役割の曖昧さ

| 問題 | 内容 |
|------|------|
| **二重表示で主従が不明** | 「お客様満足度★5.0」と「Google口コミ★5.0」が並んでおり、**同じ評価を2通りで言っている**。どちらが主でどちらが従か、あるいは「満足度＝Google口コミ」の説明関係が視覚的に伝わっていない。 |
| **伝えたいことが分散** | 満足度・Google評価・対応エリア・作業イメージを**一本の緑バーに横並び**しており、「この帯で一番伝えたいこと」がはっきりしない。 |

### 2-2. ビジュアル・色彩

| 問題 | 内容 |
|------|------|
| **色の支配とアクセントの崩れ** | 鮮やかな緑を背景に、オレンジの円・黄色の大きな数字・白文字が並ぶ。**80/15/5 やアクセント1色**のルールから外れ、**色が多すぎてまとまりがない**。 |
| **コントラスト・可読性** | オレンジ円内の白文字（お客様満足度・★5.0）は、円の色によってはコントラストが不足しうる。ラベル「Google口コミ」も小さく、緑地の白では弱い。 |

### 2-3. タイポグラフィ・スケール

| 問題 | 内容 |
|------|------|
| **★5.0 が過大** | 中央の「★5.0」が **56px（.big-num）** で極端に大きく、**ラベル「Google口コミ」の12pxと主従が逆転**している。デザイン仕様では星・評価表示は **20〜28px** を目安としており、現状は逸脱。 |
| **フォントサイズの階層が単純** | 9px / 12px / 20px / 56px など飛び飛びで、**意図した見出し・補助・本文の階層**になっていない。 |

### 2-4. 写真の扱い

| 問題 | 内容 |
|------|------|
| **「貼り付け」のまま** | 右端の作業写真が**緑背景に角や余白の処理なく直置き**してあり、**帯のデザインに組み込んだ感じが出ていない**。フレーム・オーバーレイ・角丸・余白のどれもなく、のり付け感がある。 |
| **役割の不明確さ** | この帯は「Google口コミ・信頼」がテーマなのに、**作業写真が入る必然性**（「口コミと写真の関係」）が説明されておらず、ごちゃまぜに見える。 |

### 2-5. レイアウト・余白 — 「真ん中にポツンと」の正体

| 問題 | 内容 |
|------|------|
| **中央に一点集約（チープの典型）** | 3ブロックを **flex + justify-content: center** で**画面中央に寄せ**てあり、**「真ん中にポツンと」** している。web-design-creation の **「チープ＝全部中央揃え」「プレミアム＝非対称（55:45, 60:40）」** に照らすと、このセクションは**チープ側**に入る。 |
| **横並びのだけ** | 左・中・右を並べただけで、**視線の流れ・読む順序・優先度**が設計されていない。中央一点に集約しているため、**余白と密度のコントラスト**がなく、単調で安っぽく見える。 |
| **余白が足りない** | セクション padding が **16px 20px** のみ。緑一色で幅いっぱいのため**息苦しく**、デザイン仕様の「余白のメリハリ」「コンテンツ左右は画面幅の10〜15%」を満たしていない。 |
| **装飾のバラバラ** | オレンジの conic-gradient 円、🌿 1文字、黄色の巨大数字、無加工の写真と、**パーツごとにスタイルがばらばら**で、「一つの帯」としての**共通のデザイン言語**がなく低レベルに見える。 |

### 2-6. 仕様・ルールとの齟齬

- **星・評価**: 仕様では「20〜28px」「drop-shadow 控えめ」。現状は 56px で過大、影のルールも未整備。
- **アクセント**: 「1色」が原則のところ、緑・オレンジ・黄が同列で使われている。
- **同一パターン**: この帯は「信頼・数値」を伝える**1つのセクション**として、中身の要素が多すぎて**メッセージが分散**している。

## 3. 会議での論点 — ディレクター確認事項

1. **このセクションの役割を1文で決める**  
   - 例A: 「FVの直後に、Google評価★5.0と対応エリアを一目で伝える信頼バー」  
   - 例B: 「お客様満足度（＝Google口コミ）を主役にし、エリアは補足で示す」  
   - → **主役は何か**を決め、それ以外は従にする。

2. **「お客様満足度」と「Google口コミ」の関係**  
   - 同一指標なら**1つにまとめる**（例: 「Google口コミ ★5.0」のみにして、満足度はその説明として小さい文言で）。  
   - 分けるなら**役割の違い**をはっきりさせる（例: 左は満足度・右はGoogle、など）。

3. **写真を残すか・どう組み込むか**  
   - 残す場合: **帯の一部としてデザインに組み込む**（角丸・余白・オーバーレイ・幅の制限など）。  
   - やめる場合: 信頼バーは**テキスト＋数字＋星のみ**にし、作業写真は別セクション（例: 選ばれる理由・実績）に集約する。

4. **色・スケールのルール**  
   - 背景: 緑のままなら**トーンを少し落とす**か、または帯を**白/オフホワイト＋緑アクセント**に反転する案を検討。  
   - 星・数字: **20〜28px** に収め、ラベルとの主従を正す（ラベル≦補助、数字・星＝主役だが過大にしない）。

## 4. LPスキルに基づく方針のブラッシュアップ

（web-design-creation / design-system / デザイン仕様・数値一覧 をフル活用して方針を明確化する。）

### 4-1. 「真ん中にポツンと」をやめる — レイアウト原則

| スキル上のルール | 本セクションへの適用 |
|------------------|----------------------|
| **チープ＝全部中央揃え / プレミアム＝非対称** | **中央一点に集約するレイアウトを廃止**する。コンテンツを **左寄せ** するか、**非対称のブロック割り**（例: 左 55〜60% に評価＋エリア、右 40〜45% は余白 or 控えめな補足）で「帯全体に広がる」構成にする。 |
| **余白と密度のコントラスト** | 情報を **狭い幅にまとめず**、**コンテンツ幅（max-width）を決めたうえで、左右に余白**を取る。画面幅の 10〜15% を左右マージンにし、「ポツン」ではなく **帯としての幅と余白** をはっきりさせる。 |
| **視線の流れを設計する** | 左→右の読む順序を決める。例: 「Google口コミ」ラベル → ★5.0 → 対応エリア を **1ブロックとして左寄せ** し、右側は余白またはごく薄い補足。中央揃えの **justify-content: center** は使わない。 |

### 4-2. 具体レイアウト案（実装の指針）

- **案A（左寄せ・1ブロック）**: 信頼バー内のコンテンツを **1つのまとまり** にし、**max-width: 560px 程度で左寄せ**（margin-left: 10〜15%、または padding-left で画面幅の10%確保）。右側は **余白** のまま。中央にポツンとさせない。
- **案B（非対称 60:40）**: 帯を **grid または flex** で 2 ゾーンに分ける。左 60% に「Google口コミ ★5.0」＋エリア。右 40% は空けるか、ごく控えめな装飾（テクスチャ・ノイズのみ）。コンテンツは **左ゾーン内で左寄せ**。
- **共通**: **text-align: center / justify-content: center で中央に集約しない**。**align-items: center** は縦方向のセンタリングにのみ使い、横方向は **start** または **flex-start** で左寄せ。

### 4-3. その他のブラッシュアップ（スキル準拠）

- **余白**: 上下 **24px〜32px**、左右は **画面幅の 10〜15%** をマージンに取り、コンテンツ幅を **max-width で制限**してからその中で左寄せ。
- **星・評価**: デザイン仕様「20〜28px」に合わせ **24px 前後**。ラベル 14px で主従を正す。
- **配色**: 80/15/5 とアクセント1色に寄せる。背景を **オフホワイト＋緑のアクセント帯** に反転するか、緑なら **トーンダウン**。オレンジ円は廃止し、**黄・白・緑** で統一。
- **写真**: この帯からは **外す**。信頼バーはテキスト＋星＋エリアに限定する。

## 5. 会議結論（実装に反映する方針）

- **役割の確定**: このセクションは **「Google口コミ ★5.0 と対応エリアを、FV直後に簡潔に示す信頼バー」** とする。主役は **Google口コミの評価** 1本に絞る。
- **「真ん中にポツンと」の廃止**: **中央揃え・中央集約をやめる**。コンテンツは **左寄せ** または **非対称（左にコンテンツ・右に余白）** で配置。justify-content: center は使わない。
- **情報の統合**: 「お客様満足度」と「Google口コミ」の二重表示をやめ、**「Google口コミ ★5.0」を1つ**にまとめる。必要なら「お客様満足度」は小さく補足文言で1行だけ。
- **スケールの是正**: 星・評価表示は **24px 前後** に変更。ラベル「Google口コミ」は 14px 程度で、**主従が分かる階層**にする。
- **色の整理**: 背景は **緑をややトーンダウン** するか、**白/オフホワイト＋緑のアクセント帯** に変更。オレンジ円は廃止し、**黄・白・緑で統一**する。
- **写真の扱い**: **この帯からは写真を外す**。信頼バーは「テキスト＋星＋エリア」に限定する。
- **余白・幅**: 上下 **24px〜32px**。左右は **画面幅の 10〜15%** をマージンにし、コンテンツに **max-width** を設けたうえで **左寄せ**。余白と密度のコントラストをはっきりさせる。

## 6. 実装タスク（会議後）

1. **レイアウト**: #stats の .stats-inner から **justify-content: center を削除**し、**左寄せ（justify-content: flex-start）** に変更。コンテンツに **max-width**（例: 560px）を設け、**margin-left: 10vw または padding-left: 10%** で左側に余白を確保。中央にポツンとさせない。
2. **HTML**: 「Google口コミ ★5.0」＋「奈良・大阪・京都・兵庫・滋賀対応」を主軸にした**単一ブロック**に整理。オレンジの satisfaction-circle と .stats-truck 写真を**削除**。
3. **タイポ**: .big-num を **24px 前後**、ラベルを **14px** に。主従を明確にする。
4. **背景・余白**: 背景を **オフホワイト＋緑アクセント** または **緑トーンダウン** に。padding 上下 **24px〜32px**、左右 **10〜15%** を確保。
5. **監視**: デザイン仕様「装飾要素 — 星・評価」「配色 80/15/5」「プレミアム＝非対称・中央揃え禁止」に合わせ、実装後にチェックする。
