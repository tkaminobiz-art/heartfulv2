# A案（index.html）design-critic 辛口診断

**評価日:** 2026年3月  
**対象:** ハートフルクリーン A案（index.html）  
**参照:** design-critic スキル（褒めない・具体的・Awwwards 基準）

---

## コンテキスト

| 項目 | 内容 |
|------|------|
| **ターゲット** | 遺品整理を初めて検討する遺族。悲しみ・不安の中での意思決定。奈良・近畿在住が中心。 |
| **目的** | 電話・LINE・問い合わせへのコンバージョン。信頼と「この業者なら安心」の認知。 |

## 3秒テスト

| 項目 | 評価 |
|------|------|
| **スコア** | 6 / 10 |
| **第一印象** | 「きれいだけど、どこかで見たような企業サイト」。FV の「タグ→見出し→バッジ→CTA」の流れは紋切り型。オープニングで一瞬だけ「おっ」となるが、本編に入るとパターンが読める。 |
| **離脱衝動** | 悩み（Problem）に共感すれば読み進める。共感しなければ「また同じようなサイト」で閉じる可能性あり。 |

## 5軸採点

| 軸 | スコア /20 | 判定 |
|---|----------|------|
| 美しさ | 13 | 土台はあるが、一貫性の「均一さ」がまだ残る |
| 楽しさ | 10 | アニメはあるが単調。サプライズが足りない |
| おしゃれさ | 12 | 静寂寄りだが、トレンドの「意図的な破り」が弱い |
| 惹きつける力 | 14 | 導線・権威はある。記憶に残る1点が弱い |
| 技術的実装 | 15 | レスポンシブ・a11y は押さえている。LCP要確認 |
| **合計** | **64 /100** | **C（60+）** |

**判定ランク:** C = 平均的な企業サイト。可もなく不可もなく。Awwwards 8.0 には届いていない。

---

### 美しさ [13/20] 内訳

- **配色: 3/4** — オフホワイト・緑・オレンジ・7色以内は守れている。ただし「このサイトだけの色のルール」が弱く、業界の安全色に寄っている。
- **タイポグラフィ: 3/4** — Noto Serif / Sans、見出しと本文の差はある。和文 weight 500 以下も守れている。ジャンプ率がもう一歩欲しい（見出し 56px に対し本文 17px で約 3.3 倍はギリ）。
- **余白: 3/4** — 角丸・余白のゆらぎ（--radius-card-sm/md/lg、Problem の padding 拡大）で差はついた。それでもセクション間が `clamp(80px, 10vw, 160px)` のままの箇所が多く、1箇所だけの「意図的な差」に留まっている。
- **一貫性: 2/4** — 8px グリッド・フォントは統一されているが、**section-label + section-heading + コンテンツ** のリズムがまだ多数セクションで同一。Support の左寄せは良いが、他がすべて中央で「パターン」が目立つ。
- **精度: 2/4** — 整列は取れている。一方で、サービスカードの「写真あり」と「アイコンのみ」が混在し、グリッドの見た目がやや不揃い。Bento 的な意図的な不均等ではなく「混在」に見えるリスクがある。

---

### 楽しさ [10/20] 内訳

- **マイクロインタラクション: 3/5** — ボタンのリフト・グロー、カードの translateY(-4px) はある。ただし **すべて同じ動き**。1つだけ「このサイトだけ」の反応（例: 電話ボタンだけ光の広がりが遅い等）があると差がつく。
- **アニメーションの質: 2/5** — fade-in / reveal-up が全セクションで同じ。イージングは var(--ease) で統一されているが、**種類が2つだけ**。スタガーは JS で 50ms ずつで妥当。目的のないアニメはほぼないが、「変化」が足りない。
- **サプライズ要素: 2/5** — オープニングの円形クローズは良い。本編にはスクロール連動・パララックス・キネティックタイポなどの「予想を超える体験」がほぼない。1セクションだけでもよいので「破り」が欲しい。
- **ページ遷移・スクロール体験: 3/5** — スムーススクロール・アンカーはある。視覚的リズムの変化は Support 左寄せで少し出たが、まだ単調。

---

### おしゃれさ [12/20] 内訳

- **2026年トレンド: 2/5** — ノイズオーバーレイ・静寂デザイン寄りの余白は採用。Bento UI は **均等に近いサービスグリッド** で、大小の塊のメリハリが弱い。Liquid Glass・キネティックタイポ・動的グラデーションは未採用。
- **Awwwards との距離: 2/5** — テンプレートと区別できる「記憶に残る1点」が弱い。元警察官・狩野選手・Google★5.0 は**コンテンツ**の強さであり、**デザインの独自性**としては不足。
- **エディトリアル的美意識: 4/5** — FV の 45:55 非対称、代表セクションの写真+テキスト、Support のフル幅写真+左寄せ見出しは良い。もう1セクション「画像とテキストのスケール差」や「スティッキー」があると S に近づく。
- **ブランドとビジュアルの一致: 4/5** — 礼節・信頼・透明性を、落ち着いた色・実写・丁寧なコピーで表現できている。過度に派手ではなくブランドに合っている。

---

### 惹きつける力 [14/20] 内訳

- **Above the fold: 4/5** — ベネフィット（不安を引き受ける）・権威（Google★5.0・追加料金なし・他社3日→1日）・CTA は揃っている。オープニング後に見える FV の構成は論理的。
- **ストーリーテリング: 4/5** — 共感（Problem）→ 理由（3 Reasons）→ 信頼（代表メッセージ・狩野さん）→ 行動（CTA）の導線は設計されている。感情の波に起伏があって良い。
- **CTA の訴求力: 3/5** — 電話・LINE の2本立て、テキストのみボタン（cta-band）もあり低ハードル。一方で **CTA の形がどのセクションも「横並び2ボタン」に寄りがち**。1箇所だけ縦積みや、1つだけ大きくするなどの変化があると良い。
- **独自性・メモラビリティ: 3/5** — 「元警察官が遺品整理」は強いが、**ビジュアルで1時間後に思い出す要素**が弱い。ロゴ・色・レイアウトのどれか1つが「あのサイトだけ」と語れるレベルまでは届いていない。

---

### 技術的実装 [15/20] 内訳

- **レスポンシブ: 4/5** — 768px / 600px 等でブレーク。FV のモバイルは画像上・テキスト下に切替。問題なさそうだが、375px でのタッチターゲット・フォントサイズの実機確認を推奨。
- **パフォーマンス: 3/5** — lazy loading・font-display は記載あり。LCP は画像多めのため **2.5 秒以下に収まっているか要計測**。ノイズ SVG の data URL は許容範囲。
- **アクセシビリティ: 4/5** — prefers-reduced-motion でオープニング無効化。フォーカス outline は skip にあり。コントラスト・タッチ 44px は要確認。
- **コードの意図: 4/5** — CSS 変数・セクション単位のコメント・クラス命名は一貫している。インラインスタイルは pricing の width:100% 程度で許容範囲。

## 致命的な問題

🔴 **1. セクションリズムの単調さが「テンプレート」と判断される最大要因**  
中央見出し + カード/グリッド + CTA のパターンがまだ 5 セクション以上続いている。Support の左寄せだけでは不十分。**あと 1 セクションは「見出しなし・数字だけ大きく」や「フル幅テキスト1本」など、リズムを壊す必要がある。**

🔴 **2. 「記憶に残る1点」がビジュアルにない**  
コンテンツ（元警察官・狩野さん・Google★5.0）は強いが、**デザインだけで「あのサイト」と言えるシンボルが無い**。ロゴまわり・FV の1要素・オープニングの円の使い回しなど、1つでよいので「このサイトだけ」のビジュアルフックを仕込む必要がある。

🔴 **3. CTA の並びが全セクションで同一**  
電話 + LINE の横並びが FV・cta-band・pricing サイドバー・final-cta で繰り返されている。**1箇所は「電話だけ大きく」「LINE はテキストリンク」など、形を変える**と、AI制作感がさらに減り、変換も測りやすくなる。

---

## 改善指示（優先順位順）

🔴 **1. [致命的] セクションリズムをあと1つ変える**  
- **問題:** 中央見出し + コンテンツの型が多すぎる。  
- **修正:** 例）Promise セクションを「見出しなし」にする。または「3→1日」の数字だけ大きく、その下に短文のみのブロックにして、見出しの section-label / section-heading を削除する。  
- **理由:** 同一パターン 3 回以上を避けるため。1セクションだけ「型破り」にすると、全体が意図的に見える。

🔴 **2. [致命的] ビジュアルの「記憶に残る1点」を追加する**  
- **問題:** デザインだけで語れるシンボルがない。  
- **修正:** 例）FV のバッジエリアに、1つだけ「盾」「ハート」など**このサイト用の小さなアイラスト or 写真の切り抜き**を入れる。またはオープニングの円形モチーフを、スクロール後のどこか1箇所（例: final-cta の背景）で再登場させる。  
- **理由:** Awwwards の Creativity 軸で「記憶に残る」ためには、ビジュアルのフックが1つ必要。

🔴 **3. [致命的] CTA の形を1箇所だけ変える**  
- **問題:** 電話 + LINE の横並びが繰り返し。  
- **修正:** 例）final-cta だけ「電話」を大きく 1 つメインにし、LINE は「LINEで相談」のテキストリンク（下線のみ）にする。または cta-band で「電話で相談する」をアイコンなしの 1 本柱にし、LINE はその下の小さなリンクにする。  
- **理由:** 選択肢の見せ方を変えると、離脱率・クリック率の検証がしやすく、かつ「意図的な差」になる。

🟡 **4. [重要] サービスグリッドを Bento 的にする**  
- **問題:** 写真カードとアイコンカードが同じサイズで混在し、均等に見える。  
- **修正:** 例）写真カード 1 枚を `grid-column: span 2` で大きくする。または 1 つだけ縦長（aspect-ratio: 3/4）にして、他は正方形に近くする。  
- **理由:** 不均等グリッドにすると「意図的なレイアウト」と判断され、おしゃれさのスコアが上がる。

🟡 **5. [重要] アニメーションを1種類追加する**  
- **問題:** fade-in と reveal-up の2種類だけ。  
- **修正:** 例）代表メッセージのセクションだけ、テキストを **行ごとに mask-in（左から Reveal）** にする。または Support の見出しだけ **clip-path で 1 単語ずつ表示**。  
- **理由:** 1セクションだけ動きが違うと、「楽しさ」と「おしゃれさ」のスコアが上がる。

🟡 **6. [重要] LCP の計測と対応**  
- **問題:** FV の背景画像・代表写真・Support 画像など、Above the fold の画像が多く、LCP が 2.5 秒を超える可能性がある。  
- **修正:** FV 背景に `fetchpriority="high"` を付与。代表写真または Support 画像のいずれかを `loading="eager"` にし、他は lazy のまま。可能なら WebP/AVIF 化。  
- **理由:** 技術的実装と CWV で減点されないため。

🟢 **7. [細部] 見出しのジャンプ率を少し上げる**  
- **問題:** 見出し 56px / 本文 17px で約 3.3 倍。3 倍は満たしているが、余裕はない。  
- **修正:** 例）FV の .fv__heading を `clamp(30px, 5.5vw, 60px)` にし、section-heading を `clamp(28px, 3.5vw, 44px)` に。  
- **理由:** タイポグラフィの「メリハリ」で美しさが増す。

🟢 **8. [細部] セクション間余白にもう1箇所ゆらぎを入れる**  
- **問題:** Problem だけ padding を大きくしたが、他はすべて clamp(80px, 10vw, 160px)。  
- **修正:** 例）representative の padding を `clamp(64px, 8vw, 120px) 0` に**だけ**小さくする。前後が 80–160px なので、このセクションだけ「詰まった」ように見え、リズムの差になる。  
- **理由:** 余白の「意図的な差」がもう1つあると、一貫性の中のゆらぎとして評価される。

## 唯一の良い点

**感情導線（共感→理由→信頼→行動）がきちんと設計されており、コンテンツの強さ（元警察官・狩野選手・追加料金なし・他社3日→1日）がきちんと見える位置に置けている。** デザインを「もう一押し」するだけで、C から B 以上に持ち上げられる土台はある。
