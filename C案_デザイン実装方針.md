# C案 デザイン実装方針 — 世界水準・AIっぽさ排除

**前提**: いかにもAIが生成したデザインは使わない。世界のトップデザイナーが関わっているような、こだわりあるページにする。  
**対象**: ボタン・バッジ・カード・アニメーション・各セクション。色・構成・伝えたい順番は現状維持。

---

## 1. 全体の鉄則（AIっぽさを出さない）

- **紋切り型をやめる**  
  「タグ3つ横並び → 説明1行 → CTA1つ」「薄緑の角丸にアイコン」など、他サイトと同一の型は使わない。**このLPならではの形・リズムを1つ以上入れる。**

- **均一にしすぎない**  
  角丸・余白・フォントサイズを「全部同じ数値」にしない。**意図的なゆらぎ**（4px / 8px / 12px の使い分け、padding 16px / 24px / 32px のメリハリ）をつける。

- **アイコンに逃げない**  
  汎用アウトラインアイコン（チェック・星・電話・時計）を角丸ボックスに並べただけのブロックは作らない。**文言・数字・写真・余白**で伝えることを優先する。アイコンを使うなら、**1種類のスタイルに絞り、サイト全体で「意図的な選択」に見せる。**

- **実在感を優先**  
  ストック風・AI風の「綺麗すぎる1枚」より、**実写・現場・人物**を活かす。写真の役割（共感・信頼・実績）ごとに差をつける。

---

## 2. ボタン

| 方針 | 具体 |
|------|------|
| **型を1つにしない** | メインCTA（電話・問い合わせ・LINE）ごとに、**形・サイズ・装飾を少しずつ変える**。例：電話＝角丸控えめ＋太枠、フォーム送信＝ピル型・やや大きめ、LINE＝枠線のみ or アイコン＋短文。 |
| **ホバー・フォーカス** | 色変更だけでなく、**わずかなスケール or 影の変化**（0.2s ease）を必ず付ける。キーボードフォーカス時は outline を消さず、代わりに **2px のオフセット** でアクセシビリティを確保。 |
| **禁止** | 他サイトと文言まで似た「公式LINEでつながる（無料）」風の単体ボタンのみのブロック。ここだけの文言・配置にする。 |

---

## 3. バッジ

| 方針 | 具体 |
|------|------|
| **「数」より「質」** | 複数並べる場合、**形・色・サイズに意図的な差**をつける。例：1つだけ枠線のみ、1つだけアイコンなし・数字だけ、1つだけ角丸を強く。 |
| **トンマナ** | サイトの緑・オレンジ・黄をベースに、**1色だけ濃く／1色だけ彩度を上げる**など、ルールを1つ決めて貫く。 |
| **禁止** | 4つ同じ形・同じサイズのバッジを横並びだけにすること。 |

---

## 4. カード

| 方針 | 具体 |
|------|------|
| **セクションごとにリズムを変える** | 選ばれる理由＝画像＋テキストの交互。お客様の声＝**6枚のうち1〜2枚は形を変える**（1枚だけ縦長・1枚だけ左に大きな引用線など）。作業実績＝カードのヘッダー色・ボーダーの太さで強弱。 |
| **角丸・影** | 全カード 8px 角丸・同一の box-shadow にしない。**用途で 4px / 8px / 12px を使い分け**。影は「1セクションで1種類」を基本に、他セクションでは影を弱く or なし。 |
| **余白** | padding を 16px / 20px / 24px / 32px と**セクション・役割で変える**。 |

---

## 5. アニメーション

| 方針 | 具体 |
|------|------|
| **控えめ・意図的** | スクロールで「ふわっと表示」は **1種類の動きに統一**（例：translateY + opacity、duration 0.5〜0.7s、ease-out）。あちこちでバラバラな動きにしない。 |
| **発火タイミング** | IntersectionObserver の threshold は 0.1 前後。**一度動いた要素はリセットしない**（繰り返しアニメにしない）。 |
| **禁止** | ローディング風のスピナー・過度なバウンス・複数要素が次々に流れる「テンプレ風」演出。 |

---

## 6. セクション別のこだわり

| セクション | こだわりポイント |
|------------|------------------|
| **FV（ファーストビュー）** | 主役を1つに決める（コピー or 写真のどちらかを「一番大きく・目が行く」にする）。円形ブロックだけに頼らず、**余白・タイポの階層**で視線を誘導。背景はノイズ・グラデでよいが、**1色の面積を大きくする**など「このサイトだけ」のトーンを1つ入れる。 |
| **特徴3つ（✓ 元警察官／見積無料／追加不要）** | 3つ同じ形にしない。**1つは数字・1つは短文・1つはアイコンなし**など差をつける。チェックマークをやめるか、**線の太さ・色をサイト用に1種だけ**に統一。 |
| **統計バー** | 「関西」「密着」を数字のように見せず、**実数（対応件数・満足度など）**があれば数字で見せる。満足度★5.0はそのまま活かす。 |
| **悩み（吹き出し）** | 吹き出しの形・角の尖り方を**左右で少し変える**。中央の写真は1枚に絞り、**周囲の余白で「相談」の空気**を出す。 |
| **ご安心ください（元警察官バッジ）** | ゴールド風バッジは差別化として維持。**影・縁の太さ**を他セクションと違う数値にして「ここだけのアクセント」に。 |
| **選ばれる理由（POINT 01〜05）** | 画像＋テキストの交互は維持。**POINT バッジの形**（角丸／四角／枠線のみ）を1つだけ他と変える。写真の角丸・枠の太さは統一でOK。 |
| **お約束（ポリシー5つ）** | 5枚すべて同じカード型にしない。**1つは左線のみ・1つは番号を大きく**など、リズムを変える。アイコンは廃止し、**番号＋見出し＋本文**で伝えるか、アイコンを使うなら1スタイルに統一。 |
| **CTAブロック（3回）** | 3回とも同一レイアウトにしない。**1回目＝電話強調、2回目＝LINE＋QR、3回目＝フォーム誘導**など、文言・配置に少しずつ差をつける。 |
| **作業実績（ケース01〜03）** | カードのヘッダー色・ボーダーで強弱。BEFORE/AFTER 写真はそのまま活かし、**余白・ラベルのフォント**で品位を出す。 |
| **お客様の声** | 6枚とも同じカードにしない。**1枚は引用線を大きく、1枚は星のみ・文言短め**など形を変える。アイコンは人物アイコンより**イニシャル・年代・星**だけで十分。 |
| **料金・支払い・流れ** | 表は**線の色・セル背景**をサイトの緑系統で統一。支払い方法はアイコン画像に頼らず、**文言・番号リスト**で伝えてもよい。 |
| **FAQ** | アコーディオンでなくてもよい。**Q/A の余白・左線の色**でセクションの品格を出す。 |
| **フッター** | 情報を詰め込みすぎない。**余白・フォントサイズ1段階**で締める。 |

---

## 7. 実装時のチェックリスト

- [ ] 汎用アウトラインアイコンを、角丸ボックスにそのまま並べただけのブロックにしていないか
- [ ] 角丸・余白・フォントがすべて同じ数値で均一になりすぎていないか
- [ ] ボタン・バッジ・カードに「このLPならでは」の形・リズムを1つ以上入れているか
- [ ] アニメーションは1種類の動きに統一し、過度な演出になっていないか
- [ ] 各セクションで「1つだけ形を変える」などのメリハリをつけているか
- [ ] 実写・現場・人物の写真を優先し、ストック風・AI風の綺麗すぎる1枚だけに頼っていないか

---

以上の方針で、**色・構成・伝えたい順番は維持したまま**、ボタン・バッジ・カード・アニメーション・各セクションにこだわって実装する。
