## 4. Cさん（コピーライター）の主張

### 4-1. 前提

- コピーの目的は**「読んだ人に次の行動を取らせること」**。美文でも情報羅列でもない。
- **企業主語ではなくユーザー主語**で書く。「当社は〜」ではなく「あなたは〜」「〜で困っていませんか」。
- **FAB**：Feature（機能）で止めず、**Benefit（便益）**まで翻訳する。例：「追加料金なし」→「見積り後の請求額が、1円も増えない。」
- **骨と皮膚**：見出しは**骨7：皮膚3**（事実＋余韻）。本文は骨5：皮膚5。CTA周辺は骨9：皮膚1。
- **尖りと品格**：遺品整理は**高額・デリケート**なので、**尖り3：品格7**。煽り禁止。CTA誘導文は**必ず丁寧語**。

### 4-2. ページに必須のもの

1. **見出し**  
   数字を入れる・問いかける・意外性・1メッセージのみ。40字目安。他社にも貼れる汎用表現は使わない。

2. **固有名詞・数字**  
   元警察官・5年・Google★5.0・追加料金ゼロ・他社3日→1日・奈良・近畿一円。**裏付けのない数字は使わない**。

3. **禁止フレーズ（絶対NG）**  
   「お届けします」「こだわり」「寄り添う」「お気軽に」「唯一無二」「〜という新しい選択」等。AI臭・何も言っていないコピーは信頼を損なう。

4. **引き際**  
   同じ主張を2回繰り返さない。1セクション1メッセージ。必死さが透けない「余裕」で品格を出す。

5. **CTAと誘導文**  
   ボタンラベルは低ハードル（A さんと一致）。**CTAの直前の一文は丁寧語**。「さあ、〜しませんか」は軽すぎるので使わない。

6. **ネガティブな疑問に正面から答える**  
   「後から追加料金が来ない？」→「見積り額＝最終請求額。一円の追加もありません。」と先回りして書く。不安を誠実に解消すると信頼になる。

### 4-3. 構成・デザインへの注文

- **順序**は A さんに従う。C は**各ブロックに「何を感じさせ、何をさせるか」**を言葉で設計する。
- 見出しを削ってリズムを変える場合、**そのセクションの「役割」が言葉で伝わるか**だけ確認したい。見出しがなくても、リード文や引用の一言で「ここは共感」「ここは証拠」が分かればOK。

### 4-4. 懸念（A・B への問い）

- **キャッチ**が「特徴」で止まっていないか。必ず**ベネフィット**（読み手の不安がどう解消されるか）まで書く。A さんの結論ファーストと C の FAB は一致させる。
- **権威バッジ**の文言が「当社の実績」主語になっていないか。**ユーザーにとっての意味**に変換したい（例：「Google口コミ ★5.0」→ そのまま事実でOK。「累計○件」→「○件のご家族が選んだ」）。
- デザインで**見出しを極端に小さくしたり色を弱くしたり**すると、流し読みで「骨」が拾えなくなる。B さんには見出しの**視認性**を担保してほしい。

### 5-2. FV（3者合意）

## 3. セクション別コピー設計（C リード・3者参照）

### FV
- **キャッチ**：ベネフィット＋結論。「知らない業者に任せられない。——その不安、元警察官が解決します。」（骨＋皮膚）。数字はバッジ or 副コピーで。
- **副コピー**：追加料金ゼロ・立ち会い不要・相談無料を事実で。
- **損失1行**：「後回しにすると…」1文。長くしない。
- **CTA**：「無料で話を聞いてもらう」。直下「しつこい営業・即決は一切求めません」等（丁寧語）。
- **LINE**：「状況をLINEで送るだけ」。

### Problem
- **引用**：ユーザーの心の声をそのまま。何から手をつけていいかわからない／業者を家に上げるのが怖い／後回しにしている、等。
- **損失**：「後回しにすればするほど…」「あのとき頼んでいれば」で終わる1文。押しすぎない。

### Affinity
- **各カード**：不安＝ユーザーの言葉（短く）。解消＝事実＋Benefit。例：「信頼できるか？」→「元警察官が代表。コンプライアンスは職業で証明済み。」

### Solution
- **経歴**：事実の羅列。5年・孤独死現場・独立・遺品整理士・奈良中心。
- **引用**：代表の実在の言葉。確認・承認推奨。「急かしません」「あなたのペースで」等を含む。

### Evidence
- **数値**：★5.0・0円・5年・3→1日等、裏付けあるものだけ。
- **理由**：追加料金ゼロ・立ち会い不要・買取で軽減。各2〜3行、Benefit まで。
- **声**：第三者証言のまま。改変しない。出典（Google口コミ等）を明記。

### Offer
- **サービス**：遺品整理・生前整理・特殊清掃・不用品回収。各1行 Benefit。
- **料金表**：事実のみ。注記で「他社で断られたケースも相談可」「買取で相殺」等。

### Narrowing Down
- **選ばれている**：親が亡くなった方・立ち会えない方・他社見積りが高かった方等、具体列挙。
- **ご注意**：即決のみ希望・エリア外等、正直に。

### Action
- **見出し**：「まだ決めていないで、大丈夫。話すだけでも、構いません。」（品格・寄り添い）
- **Risk Reversal**：しつこくない・即決不要・何度でも無料・断っても可・迷っている方歓迎。箇条書き・丁寧語。

## 5. 実装計画：フォント・アニメーション（B リード・3者参照）

デザインの方向性を実装で再現するため、**フォント**と**アニメーション**のルールを共通化する。

### 5-1. フォント計画

| 用途 | フォント | ウェイト | 使用箇所 | 備考 |
|------|----------|----------|----------|------|
| **本文・UI** | Noto Sans JP | 400, 500 | body、ボタン、ナビ、本文、フォーム | 和文の基準。500 は強調・ラベルに限定。 |
| **見出し・キャッチ・引用** | Noto Serif JP | 400, 500 | .section-heading、FV キャッチ、引用、Promise 数字の補足 | 品格・信頼感。ジャンプ率確保のため見出しで優先。 |
| **ラベル・数字・アクセント** | Inter | 300, 400 | .section-label、価格・件数・★、電話番号、タブular-nums | 欧文・数字の可読性。和文 weight 500 までというルールの例外として数字に使用。 |

**フォント読み込み**

- Google Fonts で **preconnect + display=swap** を使用。
- サブセット：Noto Sans JP / Noto Serif JP は `&display=swap` のまま。必要なら `&text=` で日本語常用漢字に絞って軽量化可能（要計測）。
- フォールバック：`'Noto Sans JP', 'Hiragino Sans', 'Hiragino Kaku Gothic ProN', sans-serif`（本文）、`'Noto Serif JP', 'Hiragino Mincho ProN', serif`（見出し）。

**タイポルール（再掲・実装用）**

- 見出し／本文のフォントサイズ比は **3倍以上**（例：本文 15–17px → 見出し 28–44px）。
- 数字は **font-variant-numeric: tabular-nums** を指定（Inter または Noto Sans JP で）。
- 和文は **font-weight: 500 まで**。700 は使わない（テンプレート感防止）。

**パターン別の差**

- **A・B**：上記のまま。Noto Sans JP + Noto Serif JP + Inter。
- **C**：同じフォントスタックで統一。色トークン（紺・緑CTA・ゴールド）のみ変更し、フォントは共通。

### 5-2. アニメーション計画

#### 方針

- **第一印象**：オープニングで「このサイトは丁寧」と伝え、3秒以内に結論と CTA に到達できるようにする（A 要件）。長いアニメーションで CTA を隠さない。
- **スクロール**：セクション登場は控えめに。**fade-up / mask-in** で視線を自然に誘導。同一パターン連続を避ける（B 要件）。
- **マイクロインタラクション**：ボタン・カードは **hover / focus** で 200–400ms。派手なバウンスは使わず、品格を保つ。
- **アクセシビリティ**：**prefers-reduced-motion: reduce** 時はアニメーションを無効化し、オープニングは非表示または即時表示（現行 index.html と同様）。

#### イージング・時間のトークン

| トークン | 値 | 用途 |
|----------|-----|------|
| **--ease** | cubic-bezier(0.16, 1, 0.3, 1) | 汎用（ボタン・カード・スクロール表示） |
| **--ease-out** | cubic-bezier(0.22, 1, 0.36, 1) | オープニングの fade-up・文字出現 |
| **--ease-in-out** | cubic-bezier(0.65, 0, 0.35, 1) | 円の閉じる等、終わりがゆっくりな動き |
| **duration 短** | 200–300ms | hover、色・背景の変化 |
| **duration 中** | 400–500ms | ボタン・カードの transform、アコーディオン |
| **duration 長** | 600–900ms | スクロールで見えた時の fade-up / mask-in |
| **オープニング** | 700–1600ms（要素ごとに delay） | キャッチ・サブ・CTA の順。合計 2.5 秒前後で本編表示へ。 |

※ C パターンでは **--c-ease** 等、必要に応じて同じ値で別変数名を定義してよい。

#### オープニング（A パターン推奨）

- **流れ**：グロウ → キャッチ文字（1文字ずつ or 一括）→ 区切り線 → サブコピー → CTA → 円で画面を閉じる → 本編表示。
- **キーフレーム**：op-glow / op-char / op-divider / op-fade-up / op-content-fade / op-circle-shrink（現行 index.html をベースに統一）。
- **reduce-motion**：オープニングブロックを `display: none` または中身を即時表示し、円アニメーションは行わない。

#### スクロールアニメーション

| クラス | 初期状態 | 表示時 | 時間・イージング | 用途 |
|--------|----------|--------|------------------|------|
| **.fade-in** | opacity:0, translateY(24px) | opacity:1, translateY(0) | 700ms, --ease | 汎用セクション・カード |
| **.reveal-up** | opacity:0, translateY(40px) | opacity:1, translateY(0) | 800ms, --ease | 見出し・重要ブロック |
| **.mask-in** | clip-path: inset(0 100% 0 0) | clip-path: inset(0 0 0 0) | 900ms, --ease | 「記憶に残る1点」用の差別化 |
| **stagger** | 子要素に transition-delay を 80–120ms ずつ | — | 親と同じ duration | カード3枚など、連続要素 |

- **Intersection Observer** で `.visible` を付与。閾値は 0.1–0.15、rootMargin で少し早めに発火させてよい。
- 同一セクション内で **fade-in / reveal-up / mask-in** を混在させてリズムを変える（B 要件：同一パターン3回禁止）。

#### ホバー・インタラクション

| 要素 | 変化 | 時間 |
|------|------|------|
| **.btn（プライマリ・セカンダリ）** | background, border-color, color | 200ms |
| **.card（サービス・声・料金）** | transform: translateY(-2px), box-shadow 強化 | 400ms, --ease |
| **ヘッダー（スクロール後）** | box-shadow 付与 | 400ms, --ease |
| **アコーディオン** | max-height, opacity | 400–500ms, --ease |
| **フローティング CTA** | transform（表示・非表示） | 300ms, --ease |

- **focus-visible** も同程度の transition でスタイルを当て、キーボード操作時も分かりやすくする。

#### パターン別の差

- **A（index.html）**：オープニングあり。fade-in / reveal-up / mask-in を用途に応じて使用。イージングは --ease 統一でよい。
- **B（index-b.html）**：オープニングなし or 簡易版。reveal の stagger を活かし、余白とことばのリズムに合わせる。reduce-motion 時は即時表示（現行の 0.01ms 相当）。
- **C（index-c.html）**：オープニングは簡易（fade のみ）またはなし。CVR 優先のため、長いアニメーションで CTA を遅らせない。スクロールは fade-in 中心で統一しても可。
