## 1. 計画の目的と方針

| 目的 | 内容 |
|------|------|
| **AIらしさの排除** | 紋切り型レイアウト・均一数値・安全色だけ・汎用アイコン依存を減らし、「このサイトならでは」のルールを1つ以上入れる。 |
| **デザインのブラッシュアップ** | Awwwards水準の数値ルール（余白・タイポ・色比率）を適用し、チープとプレミアムの境界を超える。 |
| **カラーリング** | 使用色を7色以内に整理し、80/15/5 を徹底。背景オフホワイト、テキストソフトブラック。 |
| **2026年トレンドの取り込み** | 現状はトレンドを押さえていないため、**静寂デザイン（Exhale）を基軸**に、採用トレンドを明示し実装する。 |

**業種との整合**: 遺品整理・生前整理は「信頼・安心・丁寧さ」が軸。参照ドキュメントの**「静寂ラグジュアリー」「ヒューマンブランド」**コンボが相性が良い。Liquid Glass・キネティック・ネオブルータリズム・ダークグローは本LPでは採用しない。

## 2. 現状診断（スキル基準での問題点）

### 2-1. Part A（デザインルール）違反・不足

| 項目 | 現状 | 問題 | 参照 |
|------|------|------|------|
| **背景色** | `--bg: #FFFFFF`、各所 `#fff` | 純白は禁止。オフホワイト（#FAFAF7, #F5F5F0）にすべき。 | A-1 色 |
| **テキスト色** | `--text: #333333` | 許容範囲だが、ソフトブラック（#1C1C1C, #222）推奨。 | A-1 色 |
| **和文 font-weight** | 見出し・タイトルに `font-weight: 700` 多数 | **和文は最大500**。700は禁止。見出しは「セリフ or 極細」で差をつける。 | A-1 タイポ |
| **border-radius 999px** | `.b-header__toll-free`、`.b-flow-cta__btn`、`.b-cta-band__bubble` | **999px禁止**。4px〜12px程度の角丸に統一。 | A-4 禁止 |
| **色比率** | 青・黄・灰・白が均等に近い | 80/15/5 が未定義。ニュートラル優位の割合にする。 | A-1 色 |
| **見出し/本文比** | 本文15px、見出しclamp(22px〜32px) | 3倍以上を確保（例: 本文15px → 見出し48px以上を1箇所は持つ）。 | A-1 タイポ |
| **セクション間** | `--section-gap: 144px` | 120〜200pxで許容。そのままでも可。 | A-1 余白 |
| **流れ6の背景** | `linear-gradient(180deg, rgba(92,107,94,0.06)...)` | 緑系の名残。プロスタッフ準拠なら青系 or ニュートラルに統一。 | 一貫性 |

### 2-2. Part B（AIっぽさ）該当

| 項目 | 現状 | 対策 |
|------|------|------|
| **均一な角丸** | 8px / 16px が大半 | 角丸を「4px（ボタン・タグ）/ 12px（カード）」など2段階にし、サイト固有ルールとして明文化。 |
| **均等グリッド連続** | 2カラム均等、4カード均等、3カード均等が続く | 1〜2セクションは非対称（例: 60:40、Bento的1枚だけ大）にする。 |
| **安全色のみ** | 青・黄・グレー・白の組み合わせのみ | 「このサイトだけのトーン」を1つ入れる（例: 暖かみのあるグレー、ブランドの薄い青味）。 |
| **紋切り型** | カード→タイトル→説明→リンク の繰り返し | セクションごとにリズムを変える（テキストのみ、数字だけ、フル幅帯など）。 |
| **CTA形状** | 角丸ボタンが複数セクションで同じ | 第1CTAは角丸、第2CTAはアウトラインのみなど、形を変える。 |

### 2-3. カラーまわり現状

- **現行変数**: primary #0066CC、primary-dark、primary-light、bg #FFF、bg-alt #F5F8FA、text #333、text-light #666、cta-yellow #FFD800、card1〜4、header-bg-right #E8F4F8。
- **問題**: 色数が多く、ニュートラル/メイン/アクセントの割合が未定義。card1〜4は青のバリエーションで、アクセントが黄以外にも分散している。

### 2-4. 2026年トレンドとのギャップ（現状が押さえていない点）

| トレンド（reference） | 現状 | ギャップ |
|-----------------------|------|----------|
| **静寂デザイン（Exhale）** | 背景は純白、見出しは太字サンセリフ均一、余白はあるが「息を吐く」設計になっていない | 背景オフホワイト・ペール、見出しは**セリフ体 or 極細**、アニメは**0.8s〜1.5s**のゆったりフェードが未適用 |
| **ベントグリッド** | 4カード・3カードはすべて均等。大小のメリハリなし | 1セクション以上で Bento（1枚だけ span 2 / tall 等）が未導入 |
| **マイクロインタラクション 2.0** | ホバーは opacity や color 変化が中心。押した感じ・浮き上がりが弱い | ボタンの **:active で scale(0.98)**、カードの **translateY(-2px) + shadow**、**scroll-reveal** が未整備 |
| **動的グラデーション・ノイズ** | グラデーションは静的なヒーロー形状のみ。ノイズなし | 「静寂ラグジュアリー」コンボとして**控えめなノイズテクスチャ** or ごく弱いグラデーションが使える |
| **人間らしさ（Imperfect）** | 整いすぎたレイアウトのみ | 「人がつくった」温かみ（わずかなテクスチャ・手書き風の線の1本など）が未活用 |
| **没入型スクロール** | 特になし | 「静寂ラグジュアリー」の軽い要素として**スクロールプログレス** or 1セクションの **sticky** が選択肢 |

**2026年の「やってはいけない」NG**（reference より）との照合:
- ❌ 真っ平らなフラット（影・アニメなし）→ 現状はヒーローに形状あり。カードの影・ホバーを強化する必要あり。
- ❌ 全セクション同じ余白・同じフォントサイズ → リズム差を計画に含める。
- ❌ ホバーが色変化のみ → リフト・スケール・押した感じを追加する。
- ❌ アニメーションが速すぎる（0.2s以下）→ 0.4s〜1.2s に統一する。

## 3. 2026年トレンドの採用方針と実装内容

### 3-1. 採用するトレンドと優先度

| 優先度 | トレンド | 採用内容 | 実装イメージ |
|--------|----------|----------|----------------|
| **必須** | **静寂デザイン（Exhale）** | 基軸。配色・余白・タイポ・アニメを Exhale に合わせる | 背景 #F7F4EF / #FAFAF7、見出しにセリフ体（Shippori Mincho）or 極細、padding 120px 級、reveal は 1.0s〜1.2s |
| **必須** | **マイクロインタラクション 2.0** | ボタン・カードの触覚的フィードバック、scroll-reveal | .haptic-btn（:hover リフト、:active scale 0.98）、.bento-card:hover 浮き上がり、.scroll-reveal + IntersectionObserver |
| **必須** | **ベントグリッド** | 1セクションで Bento 化し、均等グリッド連続を破る | 「お家の整理」4カードを Bento に（1枚 span 2、1枚 row span 2 等）。gap 14px、角丸 12〜18px、hover scale(1.02) |
| **推奨** | **ノイズテクスチャ** | 静寂ラグジュアリーの「温かみ」。控えめに | body または main に .noise-overlay::after で opacity 0.03〜0.05 のノイズ。過剰にしない |
| **推奨** | **没入スクロール（軽量）** | スクロール進捗の可視化 | 画面上部に 3px の .scroll-progress（primary のグラデーション）、width: var(--scroll-pct) |
| **任意** | **人間らしさ** | 1要素だけ「ゆらぎ」を入れる | 例: リードの見出し下に手書き風の下線（SVG）1本、または 1カードだけ rotate(0.5deg) など控えめに |

### 3-2. 採用しないトレンド（理由）

- **Liquid Glass**: テック・SaaS 向け。遺品整理LPのトーンと合わない。
- **キネティックタイポグラフィ**: クリエイティブ・アート向け。読みやすさ・信頼を優先するため見送り。
- **ネオブルータリズム**: 大胆・荒削り。安心・丁寧のブランドと相反する。
- **ダークモード・グロー**: 本LPはライト・静寂で統一する。

### 3-3. 静寂デザイン（Exhale）の具体仕様

参照: `2026-trends-reference.md` TREND 02。

| 項目 | 仕様 |
|------|------|
| **背景** | --exhale-bg: #F7F4EF または #FAFAF7、--exhale-bg-2: #EDE8E0 または #F2F2EF。セクション交互に使用。 |
| **テキスト** | --exhale-text: #2C2C2C（ソフトブラック）、--exhale-muted: #8A8480（補助）。 |
| **ボーダー** | --exhale-border: #D4CFC8。 |
| **見出しフォント** | セリフ体を採用する場合: `font-family: 'Shippori Mincho', 'Noto Serif JP', serif; font-weight: 300〜400; letter-spacing: 0.06em;`。サンセリフのままなら **weight 500 以下・letter-spacing で差**。 |
| **セクション余白** | padding: 120px 5%; max-width: 900〜1200px。 |
| **スクロールリベール** | .scroll-reveal { opacity: 0; transform: translateY(16px); transition: opacity 1.2s ease, transform 1.2s ease; }.visible で表示。threshold 0.15 程度の IntersectionObserver。 |

### 3-4. 実装するトレンド要素の一覧

1. **CSS変数**: Exhale 用の --exhale-bg, --exhale-bg-2, --exhale-text, --exhale-muted, --exhale-border を追加し、既存 --bg / --text と統合または置き換え。
2. **ノイズオーバーレイ**: main または body に .noise-overlay、::after で SVG ノイズ（opacity 0.03〜0.05）。
3. **スクロールプログレス**: .scroll-progress を fixed top、height 3px、width var(--scroll-pct)、background primary。scroll イベントで --scroll-pct 更新。
4. **スクロールリベール**: セクション見出し・カードに .scroll-reveal を付与。JS で IntersectionObserver、.visible を付与。
5. **ハプティックボタン**: .haptic-btn または主要 .btn に :hover { transform: translateY(-2px); box-shadow 強化 }、:active { transform: translateY(1px) scale(0.98) }。
6. **Bento セクション**: .b-cards-four を .bento-grid に変更し、1枚 --wide（span 2）、1枚 --tall（row span 2）等。.bento-card:hover { transform: scale(1.02) }。
7. **アニメーション時間の統一**: 0.2s 系を 0.4s 以上に。reveal は 1.0s〜1.2s、easing は cubic-bezier(0.16, 1, 0.3, 1)。

## 4. カラーリング設計（80/15/5）※Exhale と統合

### 4-1. 新パレット案（7色以内・静寂デザイン寄せ）

| 役割 | 変数名 | 色（HEX） | 用途 | 比率目安 |
|------|--------|-----------|------|----------|
| **ニュートラル・背景** | `--bg` | #FAFAF7 | メイン背景 | 80% |
| **ニュートラル・背景2** | `--bg-alt` | #F2F2EF | セクション交互・カード背景 | 同上 |
| **ニュートラル・テキスト** | `--text` | #1C1C1C | 本文・見出し | 同上 |
| **ニュートラル・補助** | `--text-light` | #5C5C5C | キャプション・補足 | 同上 |
| **ニュートラル・ボーダー** | `--border` | #E0E0DC | 線・区切り | 同上 |
| **メイン** | `--primary` | #0066CC | リンク・見出しアクセント・ヘッダー要素 | 15% |
| **メイン・濃い** | `--primary-dark` | #004C99 | ホバー・押下 | 同上 |
| **アクセント（1色のみ）** | `--accent` | #E6B800 | CTAボタンのみ（黄を少し落としたトーン） | 5% |

**Exhale との対応**: 上記のうち `--bg` を #F7F4EF にすると静寂デザインの --exhale-bg に一致する（温かみのあるオフホワイト）。`--bg-alt` を #EDE8E0 にすると --exhale-bg-2 に一致。どちらも 80% 側のニュートラルとして扱う。

**白・黒の扱い**

- 背景に純白 #FFFFFF は使わない。`#fff` 指定を `var(--bg)` または `#FAFAF7` に置換。
- テキストに真黒 #000 は使わない。`#333` は `#1C1C1C` に寄せる。
- **反転用**（青帯上の文字など）: `--text-on-dark: #FFFFFF` は維持。

**ヘッダー・ナビ**

- `--header-bg-right`: 薄い青 #E8F4F8 を維持する場合、ニュートラル側に含めて「青味のあるニュートラル」として扱う。または #F0F2F4 に寄せてよりニュートラルに。

**LINEボタン**

- 緑 #06C755 は「既存サービス色」として許容。アクセント5%には含めず、CTAのうち1種のみに限定。

### 3-2. 置換一覧（実装時）

- `--bg`: #FFFFFF → #FAFAF7  
- `--bg-alt`: #F5F8FA → #F2F2EF（青味を弱める）  
- `--text`: #333333 → #1C1C1C  
- `--text-light`: #666666 → #5C5C5C  
- `--cta-yellow`: #FFD800 → #E6B800（アクセント1色に）  
- すべての `background: #fff` / `#FFFFFF` → `var(--bg)` または `#FAFAF7`  
- カードの青（card1〜4）は `--primary` / `--primary-dark` で賄い、必要なら `--primary-light` のみ残す。

## 5. AIらしさ排除の具体施策

### 5-1. 角丸の統一ルール（このサイトだけのルール）

- **小**: 4px — タグ・バッジ・年休ボタン・入力風ボックス  
- **中**: 8px — 通常ボタン・カード・画像  
- **大**: 12px — ヒーロー画像・メディアブロック  

**禁止**: `border-radius: 999px`。該当箇所を 4px または 8px に変更。

**該当クラス**: `.b-header__toll-free`、`.b-flow-cta__btn`、`.b-cta-band__bubble` など。

### 5-2. レイアウトのリズム差（Bento で対応）

- **現状**: 2カラム均等・4カード均等・3カード均等・2カード均等が続いている。
- **施策**:  
  - **お家の整理4カードを Bento 化**（3-1・3-4 で必須）。1枚 span 2、1枚 row span 2 等で大小メリハリ。
  - 必要に応じてリードを 55:45 非対称に。または「テキストのみ・中央」セクションを1つ挟む。

### 5-3. CTAの形を分ける

- ヘッダー・ヒーロー: 塗りボタン（accent色）  
- フローCTA: アウトラインのみ（角丸 8px）、またはテキスト＋矢印のみ  
- CTAバンド内: 塗り＋LINEは現状維持で可。  
→ 「同じ角丸塗りボタン」が3回以上続かないようにする。

### 4-4. アイコン

- ヘッダーの計算機アイコンは「1セット」として許容。  
- 他に汎用アウトラインアイコン（チェック・星・時計）を角丸ボックスに並べたブロックがなければ現状のまま。  
- 追加する場合は、線の太さ・角を揃えた1セットに限定する。

## 6. デザインブラッシュアップの具体施策

### 6-1. タイポグラフィ（和文 weight 500 以下・見出しはセリフ or 極細）

- **見出し・タイトル**: `font-weight: 700` → `font-weight: 500` に一括変更。  
- **本文**: 400 のまま。  
- **数字**: Inter（または欧文）＋ `tabular-nums` は維持。  
- **差をつける**: 見出しは「サイズ・letter-spacing」で強調し、weight は 500 で統一。

**対象クラス例**:  
`.b-hero__quote`, `.b-hero__title`, `.b-cta-band__title`, `.b-cards-sec__title2`, `.b-media-sec__title`, `.b-basic-sec__title`, `.b-option-sec__title`, `.b-price-sec__title`, `.b-pillars-sec__title`, `.b-first-sec__title`, `.b-flow4-sec__title`, `.b-faq-sec__title` など、`font-weight: 700` を 500 に変更。

**2026トレンド（静寂）**: 見出しにセリフ体を入れる場合は Google Fonts で `Shippori Mincho` または `Noto Serif JP` を読み込み、`font-weight: 300〜400` で「極細」に。

### 6-2. 余白・セクション間

- `--section-gap: 144px` は 120〜200px の範囲のため維持可。  
- 見出しの上余白 > 見出しの下余白を確認（すでに `margin-bottom: 48px` 等あればそのまま）。

### 5-3. 背景の統一

- `body`, `.b-topbar`, `.b-header__left`, `.b-cert`, `.b-cta-band__card` など、`#fff` / `#FFFFFF` を `var(--bg)` に変更。  
- `.b-flow-cta` のグラデーション: 緑系をやめ、`var(--bg-alt)` への単色または `var(--bg)` → `var(--bg-alt)` のグラデーションに変更。
- 新パレットで **--exhale-bg / --exhale-bg-2** を採用している場合は、--bg / --bg-alt と同一または近い値にし、セクション交互に使い分け。

### 6-4. プレミアム感の境界（A-2）

- **避ける**: 全部中央揃え・同一パターン3回以上・均等グリッド2セクション連続。  
- **取り入れる**: 1セクションは非対称 or Bento 的変化、角丸は 4/8/12 の2〜3段階、アニメは 400〜800ms で `cubic-bezier(0.16, 1, 0.3, 1)` を維持。

## 9. 補足（トレンドとの関係・刷新後）

- **基軸**: **静寂デザイン（Exhale）**。配色・余白・タイポ・アニメを「息を吐く」設計に合わせる。
- **コンボ**: **静寂ラグジュアリー**（静寂＋動的グラデーション控えめ＋没入スクロール軽量）と **ヒューマンブランド**（人間らしさ＋静寂）の要素を、遺品整理のトーンに合わせて採用。
- **採用トレンド**: ベントグリッド（1セクション）、マイクロインタラクション 2.0、ノイズテクスチャ（控えめ）、スクロールプログレス。
- **不採用**: Liquid Glass、キネティックタイポ、ネオブルータリズム、ダーク/グロー。信頼・読みやすさ・安心を優先。
