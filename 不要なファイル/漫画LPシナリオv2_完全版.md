# 漫画LPシナリオ v2【完全版】
## タイトル：「この部屋、どうすればいいの。」

---

## ■ シナリオの核（制作前に必ず確認する）

**伝えること（1行）：**
「知らない業者が怖くて、でも一人では無理。そんなとき、元警察官が"ちゃんとした人"として現れる。」

**読者に起こさせる感情の流れ：**
①「これ、自分のことだ」→ ②「この人は信頼できる」→ ③「相談してみたい」

**登場人物は2人だけ：**
- **和子（58歳）**：遠方に住む娘。母が亡くなり一人で実家へ来た。
- **藤田（代表）**：元警察官。穏やかで誠実。説明くさいセリフは一切なし。

---

## ■ セリフの鉄則（制作前に必ず確認する）

- 1吹き出し＝**最大15文字**
- 1吹き出し＝**1つの感情だけ**
- **説明するセリフは書かない**（「私は元警察官で…」のような台詞は禁止）
- 状況はモノローグ（ナレーション枠）が説明する。セリフは感情だけ。

---

## ■ コマの読み順ルール（必須・生成AI指示にも必ず含めること）

日本の漫画は**右から左・上から下**に読む。
1ページ4コマ構成の場合、コマの配置と読む順番は以下の通り。

```
┌─────────────────────────┐
│  コマ②（左上）│  コマ①（右上）│  ← 読者はここから読み始める
├─────────────────────────┤
│  コマ④（左下）│  コマ③（右下）│  ← 次にここへ
└─────────────────────────┘

読む順番：コマ① → コマ② → コマ③ → コマ④
位置：    右上  → 左上  → 右下  → 左下
```

### 生成AIへの指示文テンプレート（各ページのプロンプトに必ず追加）

```
This is a Japanese manga page. Reading order is RIGHT to LEFT, TOP to BOTTOM.
Panel layout (2x2 grid):

- Panel 2 (second to read): TOP LEFT
- Panel 3 (third to read): BOTTOM RIGHT
- Panel 4 (fourth to read): BOTTOM LEFT

Please arrange panels accordingly so Japanese readers can follow the story naturally.
```

## ■ 誤字・文字化けを防ぐプロンプト設計（重要）

### 根本原因
AI画像生成は**日本語テキストの正確な再現が最も苦手な領域**。
特に「ナ行・ン・ソ・リ」の混同、助詞の脱落が頻発する。

### 対策3原則

**① テキストは画像に埋め込まない（最も確実な方法）**
```
手順：
1. 画像生成時は「吹き出しの形だけ」を生成し、セリフは空欄にする
2. 生成した画像をCanva・Photoshop・Affinity Designerに読み込む
3. テキストは人間が手動で入力する

プロンプト例：
「empty speech bubble, no text inside the bubble, manga panel style」
```

**② どうしても生成AIにテキストを描かせる場合のプロンプト設計**
```
・セリフを2回繰り返して明記する
  例：「セリフ内容：『急かしません』。吹き出し内のテキストは必ず『急かしません』とすること」

・1吹き出し10文字以内に絞る（文字数が多いほど誤字率が上がる）

・英語プロンプトでテキストを指定したほうが精度が上がる場合がある
  例：「speech bubble contains Japanese text: 急かしません」

・難しい漢字は使わない。ひらがな・カタカナ優先
  例：「あわてなくていいです」（「慌てなくていいです」ではなく）
```

**③ ページ単位で生成し、テキストを後から確認する**
```
・1コマずつ生成して文字を確認してから次へ進む
・一括生成はしない（誤字が増えると修正コストが跳ね上がる）
・確認したコマのみ採用。文字が崩れたコマは即再生成
```
