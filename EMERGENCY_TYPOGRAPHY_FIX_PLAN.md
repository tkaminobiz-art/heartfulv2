# 緊急会議：タイポグラフィ・LP品質 対策案

**目的**: font-size / letter-spacing の根本見直しと、折り返し・FV・画像・口コミの個別対応で、快適なUXと問い合わせを得られるLPに復旧する。

---

## 1. 根本原因：タイポグラフィ全体設計の見直し

### 1.1 現状の問題
- `--f-h2: clamp(28px, 3.6vw, 48px)` → **48px が大きすぎ**、1440px 幅でも長い見出しが折り返し（「の、」「で。」「す。」の孤立）
- `--f-lead: max 27px`、`--f-body: max 25px` → 本文・説明文も大きく、カード内・リストで折り返し事故
- `.h2` の `letter-spacing: 0.02em` が大サイズと相まって **FAQ「よくあるご質問。」で文字間が異常に広く**見える

### 1.2 対策（1440px で折り返しさせないサイズに下方修正）

| 変数 | 現状（max） | 修正案（max） | 理由 |
|------|-------------|----------------|------|
| `--f-h1` | 72px | **64px** | FVは維持でOKだが、やや抑えて他とのバランス |
| `--f-h2` | 48px | **36px** | 長文見出しが1行〜2行で収まるよう（20字前後で折れない） |
| `--f-lead` | 27px | **18px** | 説明文として読みやすく、折り返しを防ぐ |
| `--f-body` | 25px | **16px** | リスト・カード内は「説明文と同等かやや小」＝16px で統一 |

- **clamp の min** は SP で可読性を保つため現状維持かやや縮小（例: h2 min 24px、lead 14px、body 14px）。
- **見出し専用**: 折り返し防止のため `.h1`, `.h2` に `word-break: keep-all; overflow-wrap: break-word;` を付与（行末の孤立を減らす）。
- **letter-spacing**: `.h2` の `letter-spacing: 0.02em` を **0 または 0.01em** に変更し、FAQ 含め「文字間が異常に広い」印象を解消。

### 1.3 階層の維持
- FVメイン: 最大（64px 想定）
- セクション h2: FV の 56% 前後（36px）
- 説明文: h2 の 50% 前後（18px）
- リスト・カード: 説明文と同程度〜やや小（16px）

## 2. セクション別・個別対策

### 2.1 【FV】左側代表写真の初回見切れ
- **原因候補**: `min-height: 100vh` + グリッドで左列が `1.08fr` に対し、画像が `position: absolute; inset: 0` で親の高さに依存。親 `.hero-media` に明示的な高さがなく、画像が 0 高さで描画されている可能性。
- **対策**:
  - `.hero-media` に `min-height: 50vh` または `aspect-ratio: 16/9` を付与し、初回描画時から領域を確保する。
  - 画像に `width` / `height` 属性は既存のまま、CSS で `object-fit: cover` を維持。
  - 必要なら `.hero` を `grid-template-rows: 1fr` ではなく、左列に `min-height` を指定する形に変更。

### 2.2 【お悩み】チェックリストの緑強調テキスト折り返し
- **対策**: `--f-body` を 16px に下げることで「キ/ャンセル」「整/理して」の途中改行を防ぐ。
- 追加で `.trouble-list-text .hl` に `white-space: nowrap` は避け（長文で溢れるため）、`word-break: keep-all` をリスト全体に適用し、単語中での折り返しを防ぐ。

### 2.3 【お悩み】右写真のトーンミスマッチ
- **対策（コンテンツ判断）**: 現状「談笑するチーム」のまま運用し、文言で「不安に寄り添います」を補強するか、別画像差し替えはコンテンツ担当で検討。実装では **画像差し替えは行わず**、必要なら文言調整のみ。

### 2.4 【選ばれる理由】右写真が顔クローズアップ
- **対策**: `.reason-photo img` の `object-position` を **`center top` や `center 20%`** に変更し、上半身〜顔中心から「少し引いた」構図に見せる。またはアスペクトを維持したまま、コンテナ側で高さを抑え、画像の中央より上を切るようにする。

### 2.5 【ビフォーアフター】3組目の Before/After が白い
- **原因候補**: 3組目は `不用品.jpeg` / `清掃.jpeg`。パス誤りまたはファイル未配置。
- **対策**:
  - プロジェクト内で `ハートフルクリーンサイト画像/heartful画像/不用品.jpeg` と `清掃.jpeg` の存在を確認。
  - 無い場合は 1組目・2組目と同様の before-after 用画像に差し替えるか、一時プレースホルダー（alt と onerror で「画像を設置してください」表示）を入れる。

### 2.6 【口コミ】5件→3件
- **対策**: 意図が不明なため、**5件表示に復元**する。HTML で `review-card` を 5 件分確保し、2件分のコピーを追加（元の3+2構成を再現）。

### 2.7 【流れ】STEP カード内テキストの折り返し
- **対策**:
  - `--f-body` を 16px に下げ、カード内を `font-size: var(--f-body)` のままにすれば「現地訪問・お/見積り」などの割れを軽減。
  - 加えて `.flow-step-title` に `white-space: nowrap` は避け、必要なら `word-break: keep-all` で語の途中改行を防ぐ。
  - 900px 以下で `flow-steps` を 1 列にしているのは維持。PC では 5 列のまま、フォント縮小で収める。

### 2.8 【FAQ】「よくあるご質問。」の letter-spacing
- **対策**: 上記のとおり `.h2` の `letter-spacing` を **0 または 0.01em** に統一。FAQ 専用の .h2 上書きが無いか確認し、あれば削除。

### 2.9 【最終CTA】
- 修正済みのため **変更なし**。

## 6. PC表示のテキスト折り返しルール — 厳しさと他社LP参考

### 6.1 当案のルールが「厳しすぎる」か

- **現状**: 「1440px で折り返しさせない」ために h2 最大 36px・lead 18px・body 16px に下げる案を出している。
- **懸念**: これだと **PC でやや小さく感じる**「厳しい」印象になる可能性はある。一方で、**折り返し事故（「の、」「で。」の孤立）をゼロに近づける**には、コンテナ幅（例: 1072px）に対して見出しを「1行に収まる字數」で逆算する必要がある。
- **結論**: 「厳しすぎる」かどうかは **他社LPの実態** と **禁則・改行ポジションの制御** を併せて決めるのがよい。

### 6.2 他社LP・業界の参考

| 出典 | 内容 |
|------|------|
| **コナックス（日本語見出し改行）** | 静的LPでは **`<wbr>` で改行候補を手動指定** する手法を推奨。CMS では `text-wrap: balance` で行バランスを取る。日本語は単語区切りがないため、**意図した位置で折り返す**には改行候補の指定が有効。 |
| **禁則・読みやすさ** | **行頭禁則**（句読点が行頭に来ない）、**行末禁則**（開き括弧が行末に来ない）、**分割禁止**（数字と単位などを途中で切らない）が基本。ブラウザのデフォルトでもある程度かかるが、**「の、」「で。」が行頭に単独** になるのは避けたい。 |
| **1行文字数** | 本文の行長は **35〜45字（全角）** が読みやすいとされる。見出しは「1行で収める」か「意味の切れ目で2行」が多く、**見出しと本文のサイズ比 1.3〜1.6倍** という目安もある。 |
| **競合（mira1l.co.jp 等）** | 遺品整理系LPのコードでは、見出しに **2rem〜2.5rem（32〜40px）**、サブ見出しに **1.5rem〜1.8rem（24〜29px）** を使用。**48px 級の大見出しは限定的**。 |
| **デザイン仕様・数値一覧（当プロジェクト）** | 見出し h2 は **32〜48px**、本文は **16px**、リードは **14〜18px** と定義。今回の「h2 36px / lead 18px / body 16px」は **この範囲内** で、仕様とも整合する。 |

### 6.3 推奨：折り返しルールを「サイズだけ」にしない

- **A. フォントサイズの上限を抑える**（現案どおり）  
  → 1440px で無理なく収まるようにする。**他社も 40px 前後の h2 が多く、36px は厳しすぎない**。
- **B. 改行位置を制御する**  
  → 重要な見出し（料金・口コミ・流れ・FAQ など）に **`<wbr>`** を入れ、「の、」「で。」が行頭に来ないようにする。LP は静的なので **手動指定が現実的**。
- **C. 禁則を意識したコピー案**  
  → どうしても収まらない見出しは **文言を短くする** か、**改行してよい位置で区切る**（例：「あとから増える」をなくす／ための、料金設計。→ 「あとから増える」をなくすための、／料金設計。）。

**PC表示のルールとして「フォントサイズ上限」と「改行位置の指定」の両方** を置くことで、厳しすぎず・事故も減らせる。

## 7. 各スキルの所見

### デザインスキル A（ディレクター）

- **数値ルール**: デザイン仕様では見出し h2 は 32〜48px、本文 16px。**36px / 18px / 16px は仕様内**。品質基準（Awwwards を意識）からも、**折り返しで「の、」だけ行頭** は避けるべき。
- **他社参考**: 競合LPは h2 を 2〜2.5rem 程度に抑えている。**当案は他社よりやや控えめだが、業界水準から外れてはいない**。
- **所見**: 「厳しい」より **「折り返しルールを明確にした」** と捉える。加えて **`<wbr>` で改行候補を指定** すれば、サイズを 38px などに少し緩めても事故を防げる。**A としては「サイズ下方修正 + 重要見出しへの &lt;wbr&gt; 導入」を推す**。

### デザインスキル B（実装係）

- **計画との整合**: 計画書（本対策案）で「h2 36px / lead 18px / body 16px」「letter-spacing 削減」「word-break / overflow-wrap」が決まれば、**:root とコンポーネントで変数参照するだけ** で一貫実装できる。
- **&lt;wbr&gt; の扱い**: 静的 HTML なので、**見出しごとに改行してよい位置に &lt;wbr&gt; を挿入** する実装は可能。CMS でないため保守負荷も許容範囲。
- **所見**: PC の折り返しルールは **「厳しすぎる」というより「ルールがはっきりして実装しやすい」**。他社LPのように **rem で 2〜2.5rem に収めている** のと同等の考え方。**B としては現案の数値で実装し、必要に応じて &lt;wbr&gt; を追加する** 形でよい。

### デザインスキル C（監視役）

- **計画 vs 実装**: 見出し/本文比は **36px : 16px ≈ 2.25 倍**。デザイン仕様の「3倍以上」は **h1 と本文** の関係で満たせばよく、**h2 と本文は 2 倍前後で許容** されているため、監視観点では問題なし。
- **禁止事項**: 純白・真黒・和文 600 以上・letter-spacing 過剰は本対策で解消される。**折り返し事故の有無** をチェックリストに追加することを推奨。
- **所見**: 「PC で折り返しを出さない」ルールは **ユーザー体験（読みやすさ・信頼感）のためには妥当**。他社LPも大見出しを 48px にしていない。**C としては「厳しすぎる」とは考えず、ルールを明文化したうえで実装し、提出前に 1440px で見出し・リード・本文の折り返しを確認する** ことを求める。

## 8. 追補：PC表示の折り返しルール（案・文言）

以下をプロジェクトのルールとして明示する場合の文案。

- **見出し（h1 / h2）**: 1440px 幅で、**句読点や「の、」などが行頭に単独で来ない** こと。長い見出しは **&lt;wbr&gt; で改行候補を指定** するか、文言を調整する。
- **説明文（lead）**: 同上。行末禁則・行頭禁則に沿い、**不自然な 1 文字行** を避ける。
- **リスト・カード内**: **word-break: keep-all** を基本とし、**語の途中での改行** を防ぐ。フォントサイズは **16px** を上限とし、コンテナ幅に余裕を持たせる。
- **数値**: h2 最大 36px（必要なら 38px まで緩和）、lead 最大 18px、body 16px。デザイン仕様・数値一覧と矛盾しない。

このルールは **他社LPの実態（2〜2.5rem の見出し・16px 前後の本文）** および **日本語の禁則・&lt;wbr&gt; の推奨** と整合している。
