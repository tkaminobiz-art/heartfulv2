# Bパターン設計方針 v2 — Aから離れ、一枚の作品に

## 前提

- **画像は後からどんどん足していく。** 今ある素材だけで完結させず、「画像スロット」を用意し、0枚でも1枚でも4枚でもレイアウトが崩れないようにする。
- **Aに引っ張られない。** セクション順・見出しの数・情報量をAと揃える必要はない。Bは「余白とことばと数字の舞台」として独自のリズムで組む。

---

## Bのコンセプト（一言）

**「余白のなかで、約束と理由と人が、ひとつずつ現れる。」**

- 静寂（静寂デザイン）＋ 信頼（元警察官・追加料金なし）＋ 人間らしさ（ノイズ・ことば）
- 画像は「あると彩り、なくても成立する」スロット。テキストと数字が主役。

---

## 構成の自由化（Aと揃えない）

| ブロック | 役割 | 画像 | レイアウトの狙い |
|----------|------|------|------------------|
| **Hero** | 1枚の約束を印象づける | 背景1枚（差し替え可） | 全面＋中央テキスト。画像なしでも文言だけのHeroにできる余地あり。 |
| **約束（フルスクリーン）** | 一文だけの画面 | なし | 1 viewport で「追加料金は、一切いただきません。」など、1行だけ。タイポだけで見せる。 |
| **理由** | 3つを伝える | なし | 左に番号・右に文の縦リズム、または Bento。画像スロットは設けない。 |
| **人（代表）** | なぜ・誰が | **画像スロット**（1枚〜複数可） | 左テキスト・右メディア。メディアは空でも高さだけ確保。後から1枚・2枚・4枚並べられるグリッド。 |
| **料金** | 数字で伝える | なし | **意図的な破り**: 数字だけ右に大きくはみ出させる。 |
| **流れ** | 5ステップ | なし | 縦に並べる。番号をアクセントに。 |
| **声** | 口コミ | なし | 2枚＋アコーディオン。 |
| **現場・実績（任意）** | 写真で彩る | **画像スロット**（0〜N枚） | grid auto-fill。0枚なら非表示 or 余白だけ。後から足す。 |
| **FAQ** | よくある質問 | なし | アコーディオン。 |
| **CTA** | 電話・LINE | なし | 電話番号を大きく。 |
| **会社** | 概要 | なし | 表。 |

## 画像スロットのルール

1. **代表セクション**  
   - `.b-rep__media` 内に画像エリア。  
   - 子が 0 枚 → 空のブロック（min-height で高さだけ確保）。  
   - 1 枚 → 1枚大きく表示。  
   - 2枚以上 → 同じエリア内で grid などで並べられるようにする（後から足しやすいクラス名・コメント）。

2. **現場・実績（ギャラリー）**  
   - セクションごと「画像を後から足す」用。  
   - `grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))` など、枚数が増えても並ぶようにする。  
   - 0枚のときはセクションを非表示にするか、見出しだけにして「画像追加用」とコメント。

3. **Hero**  
   - 背景画像は 1 枚想定。差し替え可能にする（img の src を変えるだけ）。  
   - 画像なしで使う場合は、背景を単色 or グラデーションに切り替えられるようにしておく（class や data 属性で制御）。

---

## レイアウトの変化（「中央 720px」の繰り返しをやめる）

- **約束フルスクリーン**: 画面いっぱい、文言 1 行だけ。フォントサイズを大きく（clamp で 5vw〜12vw など）。
- **理由**: 左寄せで max-width 560px、または 番号を左に大きく（例: 左 80px / 右 1fr）。
- **代表**: 40:60 の非対称（テキスト 40、画像スロット 60）。画像がなくてもテキスト側で「誰が・なぜ」が伝わる。
- **料金**: 親に overflow: visible、価格の数字だけ margin-right: -15% や transform で右にはみ出し。
- **その他**: セクションごとに「中央」「左寄せ」「全幅見出し」を切り替え、同一パターンが3回続かないようにする。

---

## タイポの強弱

- 見出しは 1 ページに 1〜2 箇所、**極端に大きく**（例: 約束の一文だけ 8vw〜12vw）。  
- それ以外は控えめに。メリハリで「一枚の作品」感を出す。

---

## 実装時の注意

- 画像スロットには `alt=""` やプレースホルダー用の data 属性を付けておくと、後から差し替え・追加しやすい。  
- B 用の CSS は A に合わせず、クラス名も `.b-` のまま B 専用で拡張。  
- 文言・電話番号・LINE などの中身は A と共通でよいが、**並び順・見出しの数・セクションの有無は B 独自でよい**。
