# C案（ihin-seiri.html）戦略会議 — 洗い出し報告

**対象**: 遺品整理LP（C案）  
**方針**: デザインの方向性は維持しつつ「売れるLP」へ仕上げる  
**目的**: ビジュアル・コードの問題点を多角的に洗い出し、実装判断の材料とする

---

## 1. クリエイティブディレクター視点

### ビジュアルの問題点

| 項目 | 内容 | 重要度 |
|------|------|--------|
| **FV（ファーストビュー）の焦点** | ヒーロー中央の円内に「遺品整理で "お悩み" の方に」＋矢印のみで、**着地（お任せください）が下段のテキスト**。視線の流れはあるが、1秒で伝わる「ベネフィット＋CTA」の強さがやや弱い。 | 中 |
| **色の一貫性** | 緑・オレンジ・黄・クリームは統一されているが、**統計バー（#stats）の「関西」「密着」**が数字ではなく文言で大きく見せているため、説得力の「数値」が弱い。満足度★5.0は良い。 | 中 |
| **写真の役割** | 左「相談」・右「代表」は良い。**同一写真（相談.jpeg）がCTA・悩みセクション・ポリシーヘッダー等で繰り返し**使われており、リズムが単調。 | 中 |
| **信頼バッジ** | 「元警察官 代表」のゴールド風バッジ（#reassurance）は差別化として有効。他セクションとの**ビジュアルリズムのメリハリ**は取れている。 | 低 |
| **ヘッダー** | 本番LPとして**「A案のLPを見る」「B案のLPを見る」**が表示されたままだと、一般ユーザーには不要で混乱要因。 | 高 |

### コード上の問題（CD視点）

- **画像パス**: `images/` 配下（icon-estimate.png, policy-icon1.png, voice-icon1.png 等）を参照しているが、**プロジェクトルートに `images/` フォルダが存在しない**。破損・404の原因。  
- **LINE QR**: 1箇所で**ロゴ画像をLINEのQRコード代わりに表示**している（`logo-heartful-clean-darkbg.png`）。QRコード画像に差し替える必要あり。

## 2. デザイン戦略視点

### ビジュアルの問題点

| 項目 | 内容 | 重要度 |
|------|------|--------|
| **余白・タイポの数値** | セクション間は 40px〜60px 程度で安定。**見出しと本文のフォントサイズ比**はおおむね3倍以上あり。max-width 1000px 前後で統一感あり。 | 低 |
| **グリッド・背景** | grid-bg / dot-bg / ノイズテクスチャが使い分けられており、**方向性は維持してよい**。 | 低 |
| **CTAのビジュアル** | オレンジボタン・緑の電話番号・黄の「無料」バッジは認識しやすい。**同一CTAブロックが3回**（CTA1・2・3）繰り返しで、長いページでは良いが文言・レイアウトが完全同一で冗長感。 | 中 |
| **カードのリズム** | 選ばれる理由（POINT 01〜05）は画像＋テキストの交互でリズムあり。**お客様の声は6枚とも同じカード型**で、1つだけ形を変えるとメリハリが出る。 | 低 |

### コード上の問題

- **インラインスタイルの多さ**: 多くのブロックが `style="..."` で指定。**保守性・一貫性**の観点で、共通クラスとCSS変数に寄せると今後の変更が楽。  
- **レスポンシブ**: **@media が一切ない**。flex-wrap で自然に折れる部分はあるが、タブレット・スマホで**ナビ・ヒーロー・統計・料金表・フォーム**等の専用ブレークポイントがない。

## 3. UI/UXリード視点

### ビジュアル・UXの問題点

| 項目 | 内容 | 重要度 |
|------|------|--------|
| **ナビ** | 6リンク（選ばれる理由・お約束・作業実績・お客様の声・料金・流れ）は明確。**SPでは横並びのまま**で、小さい画面で押しづらい・はみ出す可能性。 | 高 |
| **電話番号のタップ** | `tel:0742268496` は設定済みで良い。**ヘッダー・CTA・フッター付近**で一貫して同じ番号。 | 低 |
| **フォーム** | お名前（必須）・郵便番号・住所・電話・メール・ご要望。**action / method 未設定**で送信先がなく、確認画面へも遷移しない。実装前提のプレースホルダー。 | 高 |
| **ページ内リンク** | `#contact` `#price` 等は smooth scroll で対応。**固定ヘッダー分のオフセット**は script で調整済み。 | 低 |
| **戻るボタン** | 「▲ 上部に戻る」が固定右下。z-index:999 で問題なし。 | 低 |
| **LINE** | 「LINE追加」リンクはあるが、**QRコードがロゴ画像のまま**の箇所があり、実際のLINE追加に支障。 | 高 |

### コード上の問題

- **IntersectionObserver**: `.point-item, .cta-box, #cases .border-green` に reveal アニメーションを付与しているが、**#cases 内のカードに `border-green` クラスが付いていない**（インラインで border 指定）。ケーススタディのカードがアニメーション対象から外れている可能性。  
- **フォーム**: `<form>` に `action` `method` `name` がなく、送信・バリデーション・確認画面の実装が未着手。

## 4. LP・転換設計視点

### ビジュアル・導線の問題点

| 項目 | 内容 | 重要度 |
|------|------|--------|
| **導線** | 悩み → 安心（元警察官）→ 選ばれる理由 → お約束 → 実績 → お客様の声 → 料金 → 支払い → 流れ → FAQ → 問い合わせ。**論理的な流れは良い**。 | 低 |
| **CTAの数** | ヘッダー「問い合わせ」＋ CTAブロック3回 ＋ フォーム。**十分な露出**。 | 低 |
| **リスクリバーサル** | 「見積り後キャンセル無料」「追加費用一切なし」は明記されており**転換に有効**。 | 低 |
| **統計の説得力** | 「Google口コミ」「★5.0」は良い。「**関西」「密着**」を大きな数字のように見せている部分は、**実数（例：対応件数・満足度）**に変えた方が説得力が増す。 | 中 |
| **お客様の声** | 「Google口コミより」表記で**参照元ルール準拠**。6件あるが、**一部が短く汎用的**（「心のこもったサービスに感謝」等）。差し替え・追記で具体性を出すとよい。 | 中 |

### コード上の問題

- **問い合わせフォーム**: 送信先・確認画面・完了画面が未実装のため、**現状では転換が完結しない**。  
- **LINE**: 実際のQRコード画像または公式LINEリンクの明示がない箇所があり、**転換経路が弱い**。

## 5. コピーライター視点

### ビジュアル・文言の問題点

| 項目 | 内容 | 重要度 |
|------|------|--------|
| **FVコピー** | 「遺品整理で "お悩み" の方に」→「お任せください」は**共感→解決**の流れで良い。「大阪・奈良・京都・兵庫・滋賀の遺品整理」は地域訴求として明確。 | 低 |
| **禁止フレーズ** | ルール（Google口コミ表記等）に沿っている。**他社名・エキテン等の表記はなし**。 | 低 |
| **主語** | お客様主語（「ご負担を減らす」「お気軽に」）が使われており適切。 | 低 |
| **具体性** | 料金例（1K 27,500円〜等）・作業実績（ケース01〜03）・支払い方法は**具体的**。 | 低 |
| **お客様の声** | 3件目「60代 男性」は本文がやや**汎用的**。「丁寧な説明とサポート」「感謝の気持ちでいっぱい」等、もう一歩具体（何が良かったか）があると説得力アップ。 | 中 |
| **FAQ** | 「24時間年中無休で受付」とあるが、他では「7:00〜22:00（土日祝も営業）」とある。**受付時間の表記が矛盾**。 | 高 |

### コード上の問題

- **表記ゆれ**: 特にコードではなく文言として「お見積り」/「お見積もり」、「ご依頼」/「お申込み」等の表記ゆれが一部ある可能性。全体の表記統一チェックを推奨。

## 6. SEO視点

### ビジュアル・構造の問題点

| 項目 | 内容 | 重要度 |
|------|------|--------|
| **見出し階層** | h1 が存在しない。**title は「遺品整理ならハートフルクリーン｜元警察官・奈良・関西対応」**で良いが、本文に h1 がなく h2 から始まっている。 | 高 |
| **セクションの意味** | 選ばれる理由・お約束・実績・声・料金・流れ・FAQ・お問い合わせと**論理的なブロック分け**はできている。 | 低 |

### コード上の問題

| 項目 | 内容 | 重要度 |
|------|------|--------|
| **meta description** | **未設定**。検索結果の説明文が出ない。 | 高 |
| **OGP** | **og:title, og:description, og:image, og:type が一切ない**。SNSシェア時に適切なプレビューが出ない。 | 高 |
| **構造化データ** | **JSON-LD（LocalBusiness / Service 等）が未実装**。リッチリザルト・知識パネルに活かせない。 | 高 |
| **タイトル** | 1本のみで問題なし。文字数も適正範囲。 | 低 |
| **画像の alt** | 多くの画像に alt あり。**アイコン・装飾の一部で alt=""** になっており、必要に応じて説明を付与するとよい。 | 中 |
| ** canonical / hreflang** | 未設定。同一サイトに他案（A/B）がある場合、canonical で C案を正規URLにするとよい。 | 中 |

## 7. コード全般の問題点（まとめ）

| 種別 | 内容 |
|------|------|
| **画像** | ・`images/` フォルダが存在しないのに、icon-estimate.png, policy-icon1〜5, voice-icon1〜6, pay-*.png, policy-doc.png 等を参照している → **404** ・LINEのQRコードがロゴ画像のままの箇所あり |
| **パス** | ・画像は `ハートフルクリーンサイト画像/...` と `images/...` の**二系統**。`images/` は要作成またはパス差し替え。 |
| **レスポンシブ** | ・**@media が無い**。スマホ・タブレット用のブレークポイント・ナビのハンバーガー化・料金表の縦並び等が未対応。 |
| **フォーム** | ・`action` `method` 未設定。送信先・確認画面・必須チェック・CSRF等の実装が未着手。 |
| **本番向け** | ・ヘッダーに「A案のLPを見る」「B案のLPを見る」が表示されたまま。C案を本番とするなら削除または非表示が無難。 |
| **FAQ** | ・「24時間年中無休」と「7:00〜22:00」の**受付時間の矛盾**を文言で修正する必要あり。 |
| **アニメーション** | ・reveal の対象に `#cases .border-green` があるが、該当要素にクラスが付与されていない。 |

## 8. 優先度サマリ（実装判断用）

| 優先度 | 項目 | 担当観点 |
|--------|------|----------|
| **P0（必須）** | 画像 404 の解消（images/ の整備またはパス変更） | デザイン・コード |
| **P0** | 問い合わせフォームの送信先・確認画面の実装方針 | UI/UX・転換 |
| **P0** | LINEのQRコードを正しい画像に差し替え | CD・転換 |
| **P0** | FAQの受付時間表記の統一（24h か 7〜22時か） | コピー |
| **P1（高）** | meta description / OGP / 構造化データの追加 | SEO |
| **P1** | h1 の付与（FVまたはタイトル相当） | SEO |
| **P1** | 本番用に A案・B案リンクを削除または非表示 | CD |
| **P1** | スマホ・タブレット用のレスポンシブ対応（@media） | デザイン・UI/UX |
| **P2（中）** | 統計バーの「関西」「密着」を数値訴求に変更するか検討 | CD・転換 |
| **P2** | 同一写真の多用の見直し（バリエーション） | CD |
| **P2** | CTAブロック3回の文言・レイアウトの微差別化 | デザイン・転換 |
| **P2** | お客様の声の一部を具体性ある文言に差し替え | コピー・転換 |
| **P3（低）** | インラインスタイルのクラス化・CSS変数整理 | デザイン・コード |
| **P3** | reveal アニメーション対象のクラス付与 | コード |
| **P3** | canonical の設定（他案との棲み分け） | SEO |

---

以上を踏まえ、**デザインの方向性を維持したまま**、上記 P0〜P1 から実装判断を進めることを推奨します。
