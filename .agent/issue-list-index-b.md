# index_b.html スクリーンショット確認　問題点一覧

## 目視で洗い出した問題点（すべて）

### 1. 3カラムレイアウトが表示されていない【重要】
- **現象**: ページ全体が単一カラムの中央寄せに見える
- **期待**: 左｜中央(750px)｜右の3カラムで、左右に色付きのサイドエリアが表示される
- **想定原因**: ビューポート幅が1024px以下でメディアクエリが効き1カラムになっている、またはグリッドが期待どおり効いていない

### 2. 左右サイドカラムの色が弱い／未適用【重要】
- **現象**: 左右のサイドカラムが存在しない、または薄い色でメリハリがない
- **期待**: 左右を**ネイビー一色**で挟み、中央コンテンツが際立つメリハリのある構成

### 3. セクション画像がフルスクリーン表示になっていない【重要】
- **現象**: 相談・遺品整理・打ち合わせ・お問い合わせなどのセクション画像が、中央コンテンツ幅（750px前後）に収まったまま
- **期待**: セクション単位の画像は画面の端から端まで（フルスクリーン／フルブリード）で表示され、メリハリが出る

### 4. 濃いネイビーセクションの扱い
- **現象**: 「なぜ元警察官だと安心できるのか？」など、濃いネイビーのセクションはあるが、これは中央コンテンツ全体の背景であり「左右のサイドカラム」ではない
- **期待**: 全セクションで「左右＝ネイビー」「中央＝コンテンツ」の枠組みが統一されている

### 5. その他（レイアウト崩れ等）
- **現象**: 画像の欠け・テキストの重なり・要素の崩れなどの明瞭なビジュアルエラーは、スクリーンショット上は見当たらない
- **備考**: レイアウト自体は崩れていないが、上記1〜4の指示が未反映であることが主な問題

---

## 対応方針

1. **3カラム＋左右ネイビー**: グリッドに依存せず、**lp-container を全幅・背景ネイビー**とし、中央に max-width: 750px のコンテンツ列を配置する方式に変更。これで「左右ネイビーで挟む」を確実に実現する。
2. **セクション画像のフルスクリーン**: `.section-hero-img` の 100vw + 負のマージンによる breakout を維持し、親の overflow を visible にし、必要なら position で確実に全幅表示にする。
3. **ブレークポイント**: 768px 以下では中央列を全幅表示し、左右ネイビーは表示しない（スマホは1カラム）。

---

## 実施した対応（完了）

- **左右ネイビー一色**: `.lp-container` の背景を `#1A3A6E` に設定。`.col-side` も `#1A3A6E` に統一。インラインの `--section-bg` は削除。
- **3カラム表示**: メディアクエリを 768px に変更（769px以上で3カラム）。`grid-template-columns: 1fr 750px 1fr` を明示。
- **セクション画像**: `.section-hero-img` に `max-width: none` と `box-sizing: content-box` を追加し、フル幅 breakout を強化。
- **問題点一覧**: 本ファイルに目視で洗い出した問題点をすべて記載済み。
