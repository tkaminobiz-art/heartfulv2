## 1. スキルによって気づいた点

- **LCP（Largest Contentful Paint）**: ヒーロー画像に `loading="lazy"` を付けると LCP が最大 35% 悪化するため、**LCP 対象には lazy を付けず、`fetchpriority="high"` を付ける**必要がある。当ページはヒーローに lazy は付いていなかったが、`fetchpriority="high"` と明示的な `width`/`height` が未設定だった。
- **CLS（Cumulative Layout Shift）**: すべての `<img>` に **`width` と `height`（または aspect-ratio）** がないと、読み込み後にレイアウトがずれる。当ページは多くの画像で寸法が未指定だった。
- **ダークモード**: §0 で「`prefers-color-scheme: dark` に対応」「背景に純黒 `#000`・テキストに純白 `#FFF` は使わない」が必須と明記されており、道Bでは未対応だった。
- **1行の文字数**: 日本語は **35〜45 文字** に収めることが推奨。PC 用の `max-width: 62ch` のままではスマホで1行が長くなりがちなため、SP では `42ch` 程度に抑える必要がある。
- **theme-color**: PWA・ブラウザ UI との一体感と、ダークモード時のアドレスバー色のために、`theme-color` を light/dark で出し分けるとよい。
- **§12 最終チェックリスト**: 提出前に「LCP 対象の lazy 有無」「全 img の width/height」「16px 以上・行間 1.8」「44px タッチ・8px 間隔」「横スクロールなし」「safe-area」「ダークモード」を一覧で確認する価値がある。

## 2. スキルによって変更した点

### §0 必守ルール 対応

| 項目 | 変更内容 |
|------|----------|
| **LCP** | ヒーロー画像に `fetchpriority="high"` と `decoding="async"`、`width="1200"` `height="675"` を追加。**loading 属性は付けない**（eager のまま）。 |
| **全 img の寸法・lazy** | 上記以外の全画像に `width`/`height` を付与。ロゴ・相談写真・和室バナー・清掃風景・ビフォーアフター 6 枚に `loading="lazy"` と `decoding="async"` を追加。清掃風景に `alt` を追加。 |
| **タイポ** | 本文は既に 16px・line-height 1.8。SP 用に `.lead` 等の `max-width` を **42ch** に制限（1行 35〜45 文字に近づける）。 |
| **タッチ** | 既存の 44px/48px 以上・gap 8px 以上を維持（前回スマホ修正で対応済み）。 |
| **レイアウト** | 既に `overflow-x: hidden`・`max-width: 100%` を適用済み。 |
| **アクセシビリティ** | 既に `prefers-reduced-motion` と `env(safe-area-inset-*)` を適用済み。 |
| **ダークモード** | `@media (prefers-color-scheme: dark)` を追加。`:root` の色を `#121212`・`#1e1e1e`・`#e8e8e8` 等に変更（**純黒・純白は使用しない**）。ヘッダー・ヒーロー・トラストバー・お悩み・安心・フッターの背景・テキストをダーク用に上書き。 |

### §12 最終チェックリスト に沿った追加

- **theme-color**: `<meta name="theme-color">` を light 用 `#fafaf7` と dark 用 `#121212` で出し分け。
- **PWA（最低ライン）**: `<meta name="apple-mobile-web-app-capable" content="yes" />` を追加。

### 変更していないが、スキルで確認した点

- ファーストビュー外の画像には `loading="lazy"` を付与済み。
- タップターゲット 44px 以上・隣接 8px 以上は既存の SP スタイルで充足。
- `prefers-reduced-motion` は既存で適用済み。
- Web フォント（Noto Sans JP）は `font-display: swap` が Google Fonts 側で設定されている想定で、CLS は `width`/`height` 指定で主に対応。
