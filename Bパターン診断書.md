# Bパターン（index-b.html）デザイン診断書

デザインスキル（Awwwards / 2026トレンド）に基づく診断。  
**結論: 現状は「モノクロ＝白黒のフラット」になっており、トレンド・余白のメリハリ・アクセントがなく、葬儀系の印象を与えている。**

---

## Step 1. スコアリング（40点満点）

| 観点 | 得点 | 根拠 |
|------|------|------|
| **ビジュアル品質** (Design 40%) | **3/10** | 白黒のみで「色の引き算」を誤解。真黒(#0A0A0A)・フル黒セクションが多く、スキルで禁止の「純白」「真黒」に近い。余白はあるが「余白と密度のコントラスト」がなく単調。アクセント1色がなく、グラデーション・グロー・ノイズなし。 |
| **ユーザビリティ** (Usability 30%) | **6/10** | 導線・レスポンシブ・可読性は一応確保。タッチターゲットは問題なし。 |
| **独自性** (Creativity 20%) | **2/10** | 2026年トレンドが未適用。Bento UI・ノイズ・ソフトグロー・キネティックタイポ・マイクロインタラクションが皆無。テンプレートと区別しづらい。 |
| **コンテンツ統合** (Content 10%) | **5/10** | 文言・画像は揃っているが、レイアウトが全セクション同一パターンで単調。 |

**総合: 16/40 → テンプレート〜平均企業サイト以下。Awwwards 8.0には届かない。**

---

## Step 2. 問題の特定（4視点 + 2026トレンド）

### ① 余白（ホワイトスペース）
- **問題**: セクション間の余白は取れているが、「余白と密度のコントラスト」がない。どこも同じ「中央ブロック・max-width」で密度が均一。
- **スキル**: 「迷ったら余白を2倍にせよ」「余白が多いだけではスカスカ。余白と密度のコントラストが重要」。

### ② リアリティ・クラフト感
- **問題**: 写真はgrayscaleのみ。ノイズテクスチャ・手書き風のあしらい・「人間らしい揺らぎ」がゼロ。AI完璧主義への反動トレンド未反映。
- **スキル**: 「あえてアナログなノイズ、ざらつき、コラージュで人間らしさを取り入れる」。

### ③ タイポグラフィ
- **問題**: 見出しと本文のジャンプ率は許容範囲。ただし数字に欧文フォント(Inter)・tabular-nums 未使用。キネティック・タイポ（行単位のマスクアニメ等）なし。
- **スキル**: 「数字: 欧文専用フォント、tabular-nums」「キネティック・タイポは2026年標準」。

### ④ 色の引き算（誤用）
- **問題**: モノクロを「黒と白だけ」と解釈。スキルでは「背景オフホワイト #FAFAF7」「テキストソフトブラック #1C1C1C」「真黒 #000 禁止」「アクセント1色を効果的に配置」「使用比率 80/15/5」。
- **現状**: 黒セクションが多すぎて重い。アクセントがなく、ソフトグロー・動的グラデーションなし → **葬儀・弔事の印象**。

### ⑤ 2026年トレンド未適用
- **Bento UI**: なし。均等グリッド・縦積みのみ。
- **クラフト感・ノイズ**: なし。
- **ソフトグロー・動的グラデーション**: なし。
- **キネティック・タイポ**: なし。
- **マイクロインタラクション**: ほぼなし（ホバーが薄いだけ）。

### ⑥ レイアウトの単調さ
- **問題**: 同一パターン（中央 max-width ブロック）の連続。非対称・Bento・スティッキー・「1箇所はみ出し」がゼロ。
- **スキル**: 「同一レイアウトパターンは1ページ2回まで」「非対称 55:45 / 60:40」「意図的な破りを1〜2箇所」。

### ⑦ アニメーション
- **問題**: スクロールイン・スタガー・reveal なし。400〜800ms の遅いアニメ・cubic-bezier・目的のある動きがなし。
- **スキル**: 「遅いアニメーション 400〜800ms」「スタガー 50ms 間隔」「スクロール連動 reveal」。

## Step 3. 改善方針（具体的な値で指示）

### 色
- 背景: オフホワイト **#FAFAF7**（純白禁止）。ダークは **#111** または **#1a1a1a** にし、フル黒セクションを減らす（黒は1〜2ブロック＋フッターに限定）。
- テキスト: ソフトブラック **#1C1C1C**。真黒 #000 禁止。
- **アクセント1色**を必ず入れる（例: ウォームアンバー **#C4851F** または ティール系）。CTA・番号・ラベル・ホバーグローのみに使用。比率は ニュートラル80% / メイン15% / アクセント5%。

### トレンドの組み込み
1. **Bento UI**: 「選ばれる理由」を不均等グリッド（01 を大きく、02・03 を横並びなど）に。
2. **ノイズ**: 全体または Hero に SVG ノイズオーバーレイ opacity 0.03〜0.05。
3. **ソフトグロー**: Hero または Message 下に radial-gradient で彩度を抑えたグロー（アンバー or グレー）。
4. **キネティック・タイポ**: Hero の h1 または Message の h2 を行単位で clip-path のスクロールイン。
5. **マイクロインタラクション**: CTA ボタンにホバーで scale(1.02) + box-shadow グロー。カードに translateY(-4px)。transition 400ms cubic-bezier(0.16, 1, 0.3, 1)。

### レイアウト
- 代表セクションを **60:40** の非対称に。
- 料金の数字だけ右に大きくはみ出させるなど「意図的な破り」を1箇所。
- 同一パターンは2回までに抑える。

### アニメーション
- セクション reveal: opacity + translateY(24px→0)、600ms、stagger 80ms。
- **prefers-reduced-motion: reduce** で無効化。

### タイポ
- 価格・番号: **Inter** + **tabular-nums** + weight 300〜400。
- 見出し: letter-spacing 0.06〜0.1em を徹底。

## 禁止事項チェック（スキル 2-8）

- 純白 #FFFFFF → **使用している（#FAFAFA は許容だが #FAFAF7 推奨）**
- 真黒 #000 → **#0A0A0A で事実上使用 → ソフトブラックに変更**
- 同一レイアウト3回以上 → **違反（全セクション同一）**
- border: 1px solid #ccc 系 → **グレー線多用 → 影・透明度に変更**
- 目的のないアニメーション → **現状はほぼなし（追加時は目的を明確に）**

## 実施した改善（index-b.html 反映内容）

- **色**: 背景をオフホワイト #FAFAF7、テキストをソフトブラック #1C1C1C、ダークセクションを #1a1a1a に変更。アクセントにウォームアンバー #C4851F を導入（ラベル・番号・CTA・ホバーに使用）。
- **ノイズ**: body に SVG ノイズオーバーレイ（opacity 0.035）を追加（クラフト感・2026トレンド）。
- **ソフトグロー**: Hero に radial-gradient のグローオーバーレイを追加。オーバーレイをグラデーションで柔らかく調整。
- **Bento UI**: 「選ばれる理由」を不均等グリッドに変更（01 をフル幅リード、02・03 を横並び）。カードは角丸・影・ホバーで translateY(-4px)。
- **マイクロインタラクション**: CTA ボタンにホバーで scale(1.02) と box-shadow グロー。カード・フロートボタンにホバー時の transform。
- **タイポ**: 価格・番号に Inter と tabular-nums。フローの番号をアクセント色に。
- **レイアウト**: 代表セクションを 1fr 1.1fr の非対称に。写真に border-radius。
- **アニメーション**: セクションに .reveal とスクロール連動の is-visible。prefers-reduced-motion で無効化。
- **その他**: ヘッダーに backdrop-filter。FAQ カードは白背景のままダークセクション上に配置。フッター・フロートを --dark に統一。
