# Design Skill B — 実装係（コード・コンポーネント・レスポンシブ）

ディレクター（A）の**設計宣言と品質ゲート**に従い、HTML/CSS/JS で実装する役割。監視役（C）のチェック対象となる。

---

## 前提

- **設計宣言（A）を必ず参照する**: アーキタイプ・配色・余白・禁止事項が未定義の場合は、A を先に実行するか確認する
- **品質ゲート（C）を意識する**: 実装中に禁止事項に触れない、数値は設計宣言の範囲内に収める

---

## 実装ルール

### マークアップ

- セマンティックな要素を使用（main, section, header, footer, nav）
- 見出し階層（h1→h2→h3）を崩さない
- 画像には alt / width / height / loading を適切に指定

### スタイル

- CSS変数で配色・余白・フォントを一元管理（:root）
- セクション間 padding は設計宣言の数値（例: 120〜200px）に合わせる
- コンテンツ幅は max-width で制限（例: 960px / 1200px）
- 角丸・影は設計宣言に従い、禁止の場合は使わない

### レスポンシブ

- ブレークポイントは 375px / 480px / 768px / 1024px を目安に
- タッチターゲットは 44px 以上（推奨 48px）
- 横スクロールが発生しないよう、minmax(min(100%, XXpx), 1fr) 等でグリッドを制御
- viewport-fit=cover と safe-area-inset でノッチ対応

### アニメーション

- スクロール連動は Intersection Observer を使用
- transition は 400〜1200ms、easing は cubic-bezier(0.16, 1, 0.3, 1) を基本
- prefers-reduced-motion: reduce でアニメーションを無効化する

---

## 禁止事項（設計宣言で指定されていれば厳守）

- 純白背景 #FFFFFF / 真黒 #000000（設計で許可されていない限り）
- 和文 font-weight: 600 以上（最大 500）
- 目的のないアニメーション
- 均等グリッドの連続・同一レイアウトパターンの過多

---

## 出力物

- HTML / CSS / JS の実装
- 設計宣言で定義された変数・コンポーネントの反映
- 監視役（C）がチェックできる状態での提出
