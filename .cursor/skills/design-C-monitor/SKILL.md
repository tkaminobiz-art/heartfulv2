# Design Skill C — 監視役（品質ゲート・禁止事項・提出前レビュー）

ディレクター（A）が設定した**品質ゲート**と**禁止事項**に基づき、実装（B）の成果をチェックする役割。提出前に必ず通過させる。

---

## 役割

- **品質ゲートの確認**: 設計宣言で定義された数値・ルールが満たされているか
- **禁止事項のチェック**: リスト化されたNGが1つも含まれていないか
- **提出前レビュー**: 実装物を読み、差分・漏れ・矛盾を指摘する

---

## 品質ゲート（例・設計宣言に合わせて調整）

| 項目 | 基準 | 確認方法 |
|------|------|----------|
| 配色 | 使用色 5〜7色以内、比率 80/15/5 前後 | :root と使用箇所を確認 |
| 余白 | セクション間 120〜200px、見出し下 48px 等 | padding/margin の値 |
| タイポ | 見出し/本文 3倍以上、和文 weight 500 以下 | font-size / font-weight |
| 背景 | オフホワイト（#FAFAF8 等）、純白禁止なら #FFF なし | background の値 |
| レスポンシブ | 横スクロールなし、タップ 44px 以上 | 実機 or DevTools |
| アニメーション | prefers-reduced-motion 対応 | CSS に reduce 時の無効化 |

---

## 禁止事項チェックリスト（設計宣言に従う）

- [ ] 純白 #FFFFFF / 真黒 #000000 を背景・テキストに使用していないか
- [ ] 和文で font-weight: 600 以上を使っていないか
- [ ] 同一レイアウトパターンが 3 回以上続いていないか
- [ ] 目的のないアニメーションがないか
- [ ] グラデーションのプレースホルダーをそのまま公開していないか
- [ ] 設計宣言で禁止された角丸・影を使っていないか

---

## 提出前レビューフロー

1. **設計宣言の有無**: A の設計宣言が存在し、実装がそれに沿っているか
2. **品質ゲート**: 上記テーブルの項目を順に確認
3. **禁止事項**: チェックリストをすべてパスしているか
4. **指摘の出力**: NG があれば箇所と修正案を明示。OK なら「品質ゲート通過」と記載

---

## 出力物

- 品質ゲートの通過/不通過と理由
- 禁止事項チェックの結果
- 修正が必要な場合の指摘リスト（ファイル名・行・内容）

---

*品質基準は design-A-director が設定し、実装は design-B-implementation が行う。本スキルは通過判定と指摘のみを行う。*
