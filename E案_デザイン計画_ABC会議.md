# E案 — デザイン計画（A・B・C 会議）

**日付**: 2026年  
**対象**: ハートフルクリーン LP の E案（デザインスキル刷新テスト）  
**役割**: A＝ディレクター / B＝実装係 / C＝監視役

---

## 1. 会議の目的

- デザインスキルを **A（ディレクター）・B（実装係）・C（監視役）** に分けたうえで、E案という**新規ビジュアル案**を1本にまとめる。
- 会議で**堅実なデザイン計画**を立て、その計画に基づいて B が実装する。C が計画と実装の整合を監視する。

---

## 2. A（ディレクター）の提案 — E案の設計方針

### 2-1. 位置づけ

- 現行 LP（静寂モダン・960px・中央揃え見出し）を**維持したまま**、E案は「**エディトリアル・やや温かみ・左揃えリズム**」で差別化する。
- コンテンツ・コピー・構成は変更しない。**ビジュアルと余白・タイポのルールだけ**を E案用に定義する。

### 2-2. 数値ルール（E案）

| 項目 | 現行 | E案 |
|------|------|-----|
| コンテンツ幅 | 960px | **1024px** |
| 背景メイン | #FAFAF8 | **#F8F7F4**（やや温かみ） |
| 背景サブ | #F5F3EF | **#F0EDE8** |
| セクション見出し | 中央揃え、margin-bottom 48px | **左揃え**、下に **2px solid var(--accent)**、margin-bottom **56px** |
| セクション間余白 | clamp(100px, 12vw, 160px) 等 | **140px** を目安に統一（clamp(100px, 12vw, 160px) は維持可） |
| ボタン角丸 | 4px | **6px** |
| カード角丸 | 8px | **6px**（ややシャープに） |
| アクセント・タイポ | 現行どおり | 変更なし（#5C6B5E、Shippori Mincho 等） |

### 2-3. 禁止事項（A の確認）

- 純白 #FFFFFF / 真黒 #000000 は使わない。
- 和文 font-weight は 500 以下。
- 同一レイアウトパターンは 3 回以上連続にしない。
- E案では「見出し左揃え＋下線」を**セクション見出しの統一パターン**として 1 回だけ適用する（他は現行維持でよい）。

---

## 3. B（実装係）の受諾と実装方針

- 上記 A の数値ルールに従い、**index-e.html** を新規作成する。
- ベースは現行 index.html とし、以下のみ差し替えする。
  - `:root` の --content-width, --bg, --bg-alt, --radius 系
  - `.section-heading` の text-align: left、border-bottom: 2px solid var(--accent)、margin-bottom: 56px
  - 該当セクションの `.section-heading` が中央揃えだった場合は left に変更
  - `.btn` の border-radius: 6px
  - カード系の border-radius を 6px に
- コピー・HTML 構造・画像・スクリプトは現行と同じにする。

---

## 4. C（監視役）の確認項目

- [ ] E案の --bg が #F8F7F4、--bg-alt が #F0EDE8 になっているか
- [ ] --content-width が 1024px になっているか
- [ ] セクション見出しが左揃え＋下線 2px・margin-bottom 56px になっているか
- [ ] ボタン・カードの角丸が 6px になっているか
- [ ] 純白・真黒・和文 weight 600 以上が使われていないか
- [ ] フッターに「E案」リンクが追加され、index-e.html に正しく遷移するか

---

## 5. 合意

- **A**: 上記計画で E案を定義した。B はこの計画に従って実装し、C は上記チェックで監視する。
- **B**: 計画に従い index-e.html を実装する。
- **C**: 実装後に上記チェックリストで確認し、問題なければ OK とする。

以上、会議による堅実なデザイン計画として確定。
