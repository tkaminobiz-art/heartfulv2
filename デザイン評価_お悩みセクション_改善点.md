# デザイン評価 — 「このような遺品整理でお悩みはありませんか？」セクション

**対象**: #trouble（お悩みセクション）  
**前提**: 美しい・読みやすい（LPスキル・デザイン仕様・design-system 準拠）  
**日付**: 2026-03-13

---

## 1. なぜ不規則に並んでいるのか — 原因の特定

### 1-1. グリッド構造による要因

現在のHTMLは **3カラムグリッド**（`grid-template-columns: 1fr 300px 1fr`）で構成されている。

| 要素 | 配置 |
|------|------|
| 左カラム（1fr） | 3つのバブル（左寄せ・`text-align: right`） |
| 中央カラム（300px） | **円形画像**（`grid-row: 1/4` で3行占有）＋ **1つのバブル**（3行目に配置） |
| 右カラム（1fr） | 3つのバブル（デフォルト左寄せ）＋ 2行目に空の `<div></div>` |

- **中央カラムだけ「画像＋バブル1つ」** の構成になっており、左右は「バブル3つ」だけ。カラムごとの**情報量・高さ・見た目の役割がバラバラ**で、**意図のない非対称**になっている。
- 2行目に **空の `<div></div>`** があり、中央列で「画像の高さ」と「バブルの有無」が混在している。その結果、**行ごとの揃いが崩れ**、不規則に見える。
- バブルに **`.left`（右揃え）と 通常（左揃え）と `text-align: center`** が混在しており、「左カラム＝右揃え」「中央＝中央」「右カラム＝左揃え」という**揃え方のルールが一貫していない**。

### 1-2. デザイン意図の欠如

- **「3列のうち中央に画像」** というレイアウトは、**視線の流れ（どこから読むか）** が定義されていない。左→右なのか、中央の画像→左右なのかが曖昧。
- バブルの**幅は 1fr に任せているだけ**で、テキスト量でセル幅が変わる。そのため **高さも幅もバラつき**、**均等グリッドでもなく、意図したBentoでもない** 中途半端な状態になっている。
- デザイン仕様の **「同一レイアウトパターン3回以上禁止」「均等グリッド2セクション連続禁止」** は、**「意図した変奏」** のためのルール。本セクションは **意図した変奏ではなく、グリッドの割り当てミス** による不規則さであり、**美しい非対称**にはなっていない。

### 1-3. まとめ（不規則になる理由）

1. **中央列だけ「画像＋バブル1つ」** で、左右と内容の型が違う。  
2. **空 div** で行の構成が崩れ、余白が意図なく抜けている。  
3. **テキスト揃え（左・中央・右）** がカラム・要素ごとにバラバラ。  
4. **バブル幅・高さ** がテキスト量に依存し、グリッドとしての「揃い」がない。  
5. **「真ん中にポツンと」**：コンテンツブロック全体が `margin: 0 auto` で中央寄せで、左右に大きな余白ができ、**帯としての幅や左寄せ・非対称** が効いていない。

## 2. スキル・仕様に基づく評価

### 2-1. デザイン仕様・数値一覧との照合

| 項目 | 仕様 | 現状 | 評価 |
|------|------|------|------|
| レイアウト | 中央一点集約禁止、左寄せ or 非対称 | 見出し・ブロックとも中央揃え、コンテンツ幅中央に集約 | ❌ |
| 余白 | 関連 8〜16px、非関連 48〜80px、左右 10〜15% | gap 24px は妥当。左右マージンは max-width のみで「帯」の余白設計なし | △ |
| カード内 padding | 32〜48px | バブル padding 14px 16px → 狭い | ❌ |
| 本文 | 16px（15〜17px） | バブル内 13px → 小さい | ❌ |
| 見出し/本文比 | 3倍以上 | h2 36px / 13px ≒ 2.8 → 不足気味 | △ |
| 配色 | オフホワイト、7色以内、アクセント1色 | 背景はグラデでオフ系。バブル #555＋白文字は可 | ○ |
| 同一パターン | 3回以上禁止（意図した変奏は可） | 7バブルが「不規則」に並んでいるだけで、変奏としての意図が伝わらない | ❌ |

### 2-2. web-design-creation / design-system との照合

| 観点 | 指摘 |
|------|------|
| **チープ vs プレミアム** | 「全部中央揃え」「均等グリッドでないのに揃っていない」状態は **チープ側**。プレミアムにするなら **意図した非対称** または **明確なグリッド＋余白** が必要。 |
| **余白と密度のコントラスト** | セクション全体が中央に寄り、左右にだらりと余白がある。**余白と密度のメリハリ** が弱く、「余白を取った設計」には見えない。 |
| **読みやすさ** | バブル内 13px は **本文 15〜17px** の仕様を下回る。カード内 padding も不足し、**読みやすさ** の条件を満たしていない。 |
| **視線の流れ** | 左・中央・右で役割が不明確で、**読む順序** が設計されていない。 |

## 3. 改善方針（美しい・読みやすいを満たすため）

### 3-1. レイアウト

- **「真ん中にポツンと」をやめる**  
  - コンテンツを **左寄せ** するか、**非対称 60:40**（例: 左にリスト、右に画像）で **意図した構成** にする。  
  - 見出しは **中央揃えのままでも可** だが、**ブロック全体** は `justify-content: center` で中央一点に集約しない。コンテンツ幅を決め、**左マージン 10〜15%** を確保したうえで左寄せ or 非対称にする。
- **不規則な3列グリッドをやめる**  
  - **案A**: **1列リスト** — バブルを縦に並べ、左寄せ。画像はセクション上部または左側に **1枚だけ** 配置（幅・高さを制限し、アスペクト比統一）。  
  - **案B**: **2列グリッド** — 左列に画像（または見出し＋リード）、右列にバブルを **2列×3〜4行** の **規則的なグリッド** で配置。バブルは **同一サイズ・同一padding** で揃える。  
  - **案C**: **非対称 40:60** — 左 40% に円形画像、右 60% にバブルを **2列グリッド**（例: repeat(2, 1fr)）で均一に並べる。中央に「1つだけはみ出す」ような配置はやめる。

### 3-2. タイポグラフィ・余白

- バブル内フォント: **15px または 16px**、行間 1.6〜1.8。  
- バブル内 padding: **24px 以上**（推奨 32px）。デザイン仕様「カード内 32〜48px」に合わせる。  
- 見出し h2: **clamp で 32〜48px** にし、本文 16px との比 **3倍以上** を満たす。

### 3-3. 揃え・一貫性

- バブル内テキストは **左揃えで統一**（`.left` の右揃え、中央の `text-align: center` をやめる）。  
- 空の `<div></div>` は **削除** し、グリッドは **「画像」と「バブルのみ」** で構成する。  
- バブルは **同一スタイル**（角丸・padding・フォントサイズ）にし、「意図した変奏」が必要なら **色や枠線の強弱** でだけ差をつける。

### 3-4. 画像

- 円形画像は **1箇所** に限定し、**役割を明確に**（「ご相談のイメージ」等）。  
- アスペクト比 **3:2 または 4:3** に統一。`border-radius: 50%` を使う場合は、サイズを **意図したサイズ**（例: 200px）に固定し、レイアウトが崩れないようにする。

---

## 4. 実装タスク（優先順）

1. **レイアウト変更**: 3列「左・中央(画像+1バブル)・右」を廃止し、**案A 1列リスト** または **案C 非対称 40:60＋2列グリッド** に変更。  
2. **HTML整理**: 中央列の空 `<div></div>` 削除。バブルのクラスを統一（`.left` や `text-align: center` をやめ、左揃えに）。  
3. **バブルスタイル**: padding **24px〜32px**、フォント **15px または 16px**、line-height 1.6〜1.8。  
4. **見出し**: h2 を 32〜48px（clamp 推奨）にし、本文比 3倍以上を確保。  
5. **セクション余白**: コンテンツブロックに **padding-left: 10%**（または margin-left）を設け、中央一点集約をやめる。  
6. **画像**: 1箇所に固定し、サイズ・角丸・アスペクト比を仕様に合わせる。
